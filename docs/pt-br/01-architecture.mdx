## Objetivo

Este componente fornece um único servidor de origem hospedando múltiplas aplicações web vulneráveis para demonstrações de testes de segurança. Ele representa a "origem" em uma arquitetura típica de balanceador de carga -- o servidor de conteúdo backend que um balanceador de carga HTTP F5 XC protege.

Em arquiteturas de produção:

```
Usuário Final -> F5 XC HTTP LB (WAF/Bot/Segurança de API) -> Servidor de Origem -> Aplicação
```

Este componente substitui um servidor de aplicação de produção real por uma VM construída especificamente, executando aplicações vulneráveis conhecidas que acionam regras de WAF, políticas de segurança de API e detecção de bots.

## Arquitetura

```mermaid

    NGINX --> |/| DEFAULT[Página Inicial]
    NGINX --> |/juice-shop/| JS[Juice Shop ×4<br/>Portas 3001-3004<br/>hash cookie_token · cache]

    NGINX --> |/vampi/| VAMPI[VAmPI ×4<br/>Portas 5101-5104<br/>ip_hash · gunicorn]
    NGINX --> |/httpbin/| HTTPBIN[httpbin ×4<br/>Portas 8201-8204<br/>gunicorn -w 4]
    NGINX --> |/whoami/| WHOAMI[whoami ×4<br/>Portas 8082-8085]
    NGINX --> |/csd-demo/| CSD[CSD Demo ×4<br/>Portas 5001-5004<br/>ip_hash · gunicorn -w 1]

    XCHLB --> |:8888| CRAPI[crAPI ×7 microsserviços<br/>Porta 8888<br/>web · identity · community<br/>workshop · postgres · mongo · mailhog]
    DVWA --> DB[(MariaDB 10.11<br/>dvwa-db)]
    REST --> RESTDB[(PostgreSQL 15.4<br/>restaurant-db)]
```

**41 contêineres** em uma VM Standard_D16s_v3 (16 vCPU, 64 GiB RAM, 60 GiB de disco).

O proxy reverso nginx:

- **Escuta na porta 80** com `reuseport` e `backlog=4096` para tráfego CDN de alta concorrência
- **Roteia por prefixo de caminho** para pools de upstream balanceados (4 instâncias por aplicação)
- **Sessões fixas** previnem perda de estado: `hash $cookie_token` para Juice Shop, `hash $cookie_PHPSESSID` para DVWA, `ip_hash` para VAmPI e CSD Demo (estado SQLite/em memória por instância)
- **Cache de proxy** para ativos estáticos do Juice Shop (zona de 10 MB, máximo de 100 MB, TTL de 60 s)
- **Registro de acesso desabilitado** para prevenir esgotamento de disco durante testes de carga CDN (logrotate como defesa em profundidade)
- **Repassa cabeçalhos do cliente** (`X-Real-IP`, `X-Forwarded-For`, `X-Forwarded-Proto`) para visibilidade na origem
- **Ajuste de kernel** via sysctl: `somaxconn=65535`, `tcp_tw_reuse=1`, `ip_local_port_range=1024-65535`

## Design Modular de Componentes

Este é uma peça de um ambiente de laboratório maior. Cada componente é autocontido e implantado independentemente:

- **Este componente** fornece o servidor de origem (nginx + contêineres Docker em VM Azure)
- **Simulador de CDN** fornece a camada de borda CDN (cache nginx em VM Azure)
- **Outros componentes** fornecem a configuração F5 XC, DNS, políticas de WAF, segurança de API, etc.

O operador humano adiciona componentes um de cada vez. A documentação de cada componente é escrita para que um assistente de IA possa lê-la e implantar a infraestrutura de forma autônoma.

## Por Que Estas Aplicações

| Aplicação | Por Que Foi Selecionada |
|---|---|
| **Juice Shop** | Projeto flagship do OWASP; SPA moderna em Node.js com mais de 100 desafios cobrindo o OWASP Top 10; mantida ativamente; 4 instâncias com cache de proxy |
| **DVWA** | Padrão da indústria para testes de WAF; níveis de segurança ajustáveis (baixo/médio/alto/impossível); build customizada com php-fpm + nginx para desempenho; backend compartilhado MariaDB 10.11 |
| **VAmPI** | Construída especificamente para o OWASP API Security Top 10; API REST com vulnerabilidades conhecidas; gunicorn com 4 workers por instância; sessão fixa ip_hash para consistência do SQLite |
| **httpbin** | Serviço canônico de testes HTTP de Kenneth Reitz; gunicorn com 4 workers gevent; útil para demos de API e inspeção de requisições |
| **whoami** | Servidor de eco de requisições do Traefik; mostra detalhes completos da requisição como a origem os enxerga -- essencial para verificar a injeção de cabeçalhos do F5 XC |
| **CSD Demo** | Página de checkout personalizada com 5 ataques estilo Magecart alternáveis (skimmer de cartão, formjacker, keylogger, criptominerador, sequestro de DOM); endpoint de exfiltração + painel do atacante; gunicorn com worker único para persistência de estado em memória |
| **DVGA** | Damn Vulnerable GraphQL Application; vulnerabilidades específicas de GraphQL incluindo injeção, DoS, ataques de batching e bypass de autorização; interface GraphiQL para exploração interativa; sessão fixa ip_hash para SQLite por instância |
| **RESTaurant** | Damn Vulnerable RESTaurant API Game; construída especificamente para o OWASP API Security Top 10 2023; FastAPI com Swagger UI; backend compartilhado PostgreSQL 15.4; cobre BOLA, BFLA, atribuição em massa, SSRF e injeção |
| **crAPI** | OWASP Completely Ridiculous API; arquitetura de 7 microsserviços cobrindo BOLA, BFLA, atribuição em massa, SSRF, manipulação de JWT e injeção NoSQL; porta dedicada 8888 (SPA com caminhos de API hardcoded); MailHog para captura de e-mail |
