## المتطلبات

### اشتراك Azure

اشتراك Azure نشط مع صلاحية إنشاء:

- مجموعات الموارد
- الشبكات الافتراضية والشبكات الفرعية
- مجموعات أمان الشبكات
- عناوين IP العامة
- الأجهزة الظاهرية (Standard_D16s_v3 -- 16 vCPU، 64 جيبيبايت RAM لأحمال العمل المكونة من 41 حاوية)

### Azure CLI

قم بتثبيت Azure CLI والمصادقة عليه:

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

### Terraform

يُشترط استخدام Terraform >= 1.5:

```bash
terraform version
```

إذا لم يكن مثبتًا، استخدم مستودع HashiCorp APT:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### صلاحيات Azure AD

يجب أن يمتلك حساب Azure AD الخاص بك صلاحية `User.Read`. يُشتق معرّف المنشئ (المستخدم في تسمية الموارد) تلقائيًا من ملف تعريف Azure AD الخاص بك. لمصادقة كيان الخدمة أو الهوية المُدارة، قم بتعيين متغير Terraform الخاص بـ `deployer` بشكل صريح.

### زوج مفاتيح SSH

زوج مفاتيح SSH للوصول إلى الجهاز الظاهري:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/origin-server-key -N ""
```

## موارد Azure التي يتم إنشاؤها

| المورد | الطراز |
|---|---|
| جهاز ظاهري Ubuntu 24.04 | Standard_D16s_v3 (16 vCPU، 64 جيبيبايت) |
| عنوان IP عام | قياسي، ثابت |
| قرص نظام التشغيل | 60 جيبيبايت Premium SSD |
| VNet + NSG | افتراضي |

راجع [حاسبة أسعار Azure](https://azure.microsoft.com/en-us/pricing/calculator/) للاطلاع على التكاليف الحالية. استخدم `terraform destroy` عندما لا يكون المختبر قيد الاستخدام لإيقاف الرسوم. راجع [إجراء التفكيك](../07-teardown/) للاطلاع على الخطوات.

:::note
يُشترط استخدام Standard_D16s_v3 (64 جيبيبايت RAM) لنشر 41 حاوية. تخصيص ذاكرة الحاويات: 4 نسخ من Juice Shop بمقدار 1 جيبيبايت لكل منها (4 جيبيبايت)، و4 نسخ من VAmPI بمقدار 512 ميبيبايت لكل منها (2 جيبيبايت)، و4 نسخ من DVWA-FPM + MariaDB (1.8 جيبيبايت)، و4 نسخ من httpbin بمقدار 256 ميبيبايت (1 جيبيبايت)، و4 نسخ من CSD Demo بمقدار 256 ميبيبايت (1 جيبيبايت)، و4 نسخ من whoami بمقدار 64 ميبيبايت (256 ميبيبايت)، و4 نسخ من DVGA بمقدار 256 ميبيبايت لكل منها (1 جيبيبايت)، و4 نسخ من RESTaurant + PostgreSQL (1.5 جيبيبايت)، و7 خدمات مصغّرة من crAPI (2 جيبيبايت)، بالإضافة إلى nginx ونظام التشغيل وذاكرة التخزين المؤقت. يبلغ إجمالي تخصيص الحاويات نحو 14.5 جيبيبايت (22.7% من 64 جيبيبايت).
:::

:::note
تسمح مجموعة أمان الشبكة (NSG) بحركة المرور الواردة على المنفذين 80 (HTTP) و8888 (crAPI). يُعدّ المنفذ 8888 ضروريًا لأن crAPI تطبيق أحادي الصفحة يُضمّن مسارات API الخاصة به بشكل ثابت ولا يمكن تقديمه خلف بادئة مسار.
:::
