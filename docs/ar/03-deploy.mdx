جميع ملفات Terraform موجودة في مجلد [`terraform/`](https://github.com/f5xc-salesdemos/origin-server/tree/main/terraform). قم باستنساخ المستودع والنشر مباشرة:

```bash
git clone https://github.com/f5xc-salesdemos/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# عدّل terraform.tfvars بمعرّف اشتراك Azure الخاص بك ومسار مفتاح SSH
```

## تهيئة Terraform

### هيكل ملفات Terraform

يحتوي مجلد terraform على 9 ملفات تتبع معيار الموارد التجريبية:

- `versions.tf` — قيود إصدارات Terraform ومزودي الخدمة (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — تهيئة مزود Azure RM و Azure AD
- `data.tf` — مصادر بيانات Azure AD للحل التلقائي للناشر
- `locals.tf` — حل الناشر، تسمية الموارد وفق إطار اعتماد السحابة من Azure، العلامات القياسية
- `main.tf` — مجموعة الموارد (بتسمية `rg-origin-server-{environment}-{deployer}`)
- `variables.tf` — جميع متغيرات الإدخال (1 مطلوب، 8 اختيارية)
- `network.tf` — شبكة افتراضية (10.200.0.0/16)، شبكة فرعية، عنوان IP عام، مجموعة أمان الشبكة (المنافذ 22/80/443/8888)، واجهة شبكة
- `vm.tf` — جهاز افتراضي Ubuntu 24.04 مع cloud-init عبر templatefile()
- `outputs.tf` — 25 مخرجاً (15 قياسياً + 10 عناوين URL للتطبيقات الخاصة بالمكونات)

`main.tf` يحتوي فقط على مجموعة الموارد:

```hcl file=./_data/main.tf
```

`variables.tf` يحدد جميع المعاملات القابلة للتهيئة. يتم حل معرّف `deployer` تلقائياً من حساب Azure AD الخاص بك — تحتاج فقط إلى تعيين `subscription_id`. القيمة الافتراضية لـ `vm_size` هي `Standard_D16s_v3` (16 وحدة معالجة مركزية افتراضية، 64 جيبيبايت ذاكرة) بحجم مناسب لـ 41 حاوية Docker:

```hcl file=./_data/variables.tf
```

### البنية التحتية للشبكة

`network.tf` ينشئ الشبكة الافتراضية، والشبكة الفرعية، وعنوان IP العام، ومجموعة أمان الشبكة (المنافذ 22/80/443/8888)، وواجهة الشبكة. المنفذ 8888 مطلوب لتطبيق crAPI الذي يعمل على منفذ مخصص:

```hcl file=./_data/network.tf
```

### الجهاز الافتراضي مع Cloud-Init

`vm.tf` ينشئ جهاز Ubuntu 24.04 الافتراضي ويمرر سكريبت التهيئة cloud-init. قرص SSD المتميز بسعة 60 جيبيبايت يوفر مساحة كافية لصور Docker وأحجام الحاويات:

```hcl file=./_data/vm.tf
```

### تهيئة Cloud-Init

`cloud-init.yaml` يهيئ الجهاز الافتراضي بتكوين محصّن. مكتبة مساعدة مشتركة (`/usr/local/lib/cloud-init-helpers.sh`) توفر منطق إعادة المحاولة لجميع عمليات الشبكة وتسجيل التقدم في `/var/log/cloud-init-progress.log`. يحل استقصاء فحص السلامة النشط محل فترات الانتظار الثابتة لجاهزية الحاويات. يقوم بتهيئة:

- **ضبط النواة**: TCP keepalive، إعادة استخدام TIME_WAIT، 2M tw_buckets، 64K somaxconn
- **nginx**: 8 كتل upstream مع جلسات ثابتة (ip_hash, cookie hash)، مجمعات keepalive (64--128)، ذاكرة تخزين مؤقت للبروكسي لـ Juice Shop، مستوى error_log الحرج
- **Docker Compose**: 41 حاوية عبر 9 تطبيقات مع حدود وحدة المعالجة المركزية/الذاكرة لكل حاوية
- **بناءات مخصصة**: DVWA-FPM (php-fpm مع pm.max_requests=200)، CSD Demo (Flask + gunicorn)، RESTaurant (مستنسخ من GitHub)
- **إعادة تدوير العمليات**: uvicorn --limit-max-requests 200 (RESTaurant)، pm.max_requests 200 (DVWA-FPM) لمنع تسرب الذاكرة تحت الحمل المستمر

```yaml file=./_data/cloud-init.yaml
```

### المخرجات

`outputs.tf` يكشف 25 مخرجاً وفقاً لمعيار الموارد التجريبية — 15 مخرجاً قياسياً مشتركاً بين جميع الموارد التجريبية (`deployer`، `public_ip`، `private_ip`، `ssh_command`، `resource_group_name`، `vm_name`، `nsg_name`، `vnet_name`، `subnet_id`، `component`، `environment`، `resource_group_id`، `vm_id`، `nsg_id`، `location`) بالإضافة إلى 10 عناوين URL للتطبيقات الخاصة بالمكونات (`origin_url`، `health_check_url`، `juice_shop_url`، `dvwa_url`، `vampi_url`، `httpbin_url`، `whoami_url`، `dvga_url`، `restaurant_url`، `crapi_url`):

```hcl file=./_data/outputs.tf
```

### مثال على ملف المتغيرات

انسخ `terraform.tfvars.example` إلى `terraform.tfvars` واملأ القيم الخاصة بك. ملف `.gitignore` يستثني `terraform.tfvars` لمنع إرسال بيانات الاعتماد:

```hcl file=./_data/terraform.tfvars.example
```

## النشر

```bash
cp terraform.tfvars.example terraform.tfvars
# عدّل terraform.tfvars بمعرّف اشتراك Azure الخاص بك ومسار مفتاح SSH

terraform init

terraform plan

terraform apply
```

يعرض Terraform عنوان IP العام وأمر SSH وعناوين URL للتطبيقات بعد النشر الناجح.

## ما بعد النشر

بعد اكتمال `terraform apply`، انتظر من 5 إلى 10 دقائق حتى ينتهي cloud-init من تثبيت Docker وسحب صور الحاويات وتهيئة nginx. عمليات سحب صور Docker هي العائق الرئيسي -- صورة Juice Shop وحدها تبلغ حوالي 400 ميبيبايت.

تحقق من نقطة نهاية الصحة:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

الاستجابة المتوقعة:

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### إعداد قاعدة بيانات DVWA

يتطلب DVWA تهيئة قاعدة البيانات لمرة واحدة. افتح `http://<PUBLIC_IP>/dvwa/setup.php` في المتصفح وانقر على **Create / Reset Database**. بيانات الاعتماد الافتراضية هي `admin` / `password`.

:::note
يتم تهيئة قاعدة بيانات SQLite الخاصة بـ VAmPI تلقائياً بواسطة سكريبت نقطة دخول gunicorn. تقوم كل حاوية بتشغيل `db.create_all()` عند بدء التشغيل، لذا لا حاجة لأي تدخل يدوي.
:::

## الربط بالمكونات اللاحقة

بعد النشر، تستخدم المكونات الأخرى مخرجات خادم الأصل كمدخلات لها:

```bash
# CDN Simulator — مرر عنوان IP العام لخادم الأصل كمصدر upstream
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# Traffic Generator — مرر اسم النطاق الكامل لموازن حمل F5 XC (وليس عنوان IP للأصل مباشرة)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| المخرج | المكون اللاحق | متغير الإدخال |
|---|---|---|
| `public_ip` | CDN Simulator | `origin_server` (مع بادئة `http://`) |
| `public_ip` | CDN Simulator | `origin_host` (مع لاحقة `:80`، بدون مخطط) |
| `public_ip` | F5 XC Origin Pool | عنوان خادم الأصل |

انتقل إلى [التطبيقات](../04-applications/) لأدلة الاستخدام أو [التحقق](../05-verify/) لاختبارات الفحص السريع.
