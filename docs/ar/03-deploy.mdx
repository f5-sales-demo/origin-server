جميع ملفات Terraform موجودة في دليل [`terraform/`](https://github.com/f5xc-salesdemos/origin-server/tree/main/terraform). قم باستنساخ المستودع والنشر مباشرة:

```bash
git clone https://github.com/f5xc-salesdemos/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# قم بتحرير terraform.tfvars بمعرف اشتراك Azure ومسار مفتاح SSH الخاص بك
```

## تكوين Terraform

### هيكل ملفات Terraform

يحتوي دليل terraform على 9 ملفات تتبع معيار الموارد التجريبية:

- `versions.tf` — قيود إصدارات Terraform والموفرين (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — تكوين موفر Azure RM وAzure AD
- `data.tf` — مصادر بيانات Azure AD للحل التلقائي للناشر
- `locals.tf` — حل الناشر، تسمية الموارد وفق إطار تبني السحابة من Azure، العلامات القياسية
- `main.tf` — مجموعة الموارد (باسم `rg-origin-server-{environment}-{deployer}`)
- `variables.tf` — جميع متغيرات الإدخال (1 مطلوب، 8 اختيارية)
- `network.tf` — شبكة VNet (10.200.0.0/16)، شبكة فرعية، عنوان IP عام، مجموعة أمان الشبكة NSG (المنافذ 22/80/443/8888)، واجهة الشبكة NIC
- `vm.tf` — آلة افتراضية Ubuntu 24.04 مع cloud-init عبر templatefile()
- `outputs.tf` — 25 مخرجًا (15 قياسيًا + 10 عناوين URL خاصة بتطبيقات المكونات)

يحتوي `main.tf` على مجموعة الموارد فقط:

```hcl file=../../terraform/main.tf
```

يحدد `variables.tf` جميع المعاملات القابلة للتكوين. يتم حل معرف `deployer` تلقائيًا من حساب Azure AD الخاص بك — تحتاج فقط إلى تعيين `subscription_id`. القيمة الافتراضية لـ `vm_size` هي `Standard_D16s_v3` (16 وحدة معالجة مركزية افتراضية، 64 جيبي بايت ذاكرة) مصممة لتشغيل 41 حاوية Docker:

```hcl file=../../terraform/variables.tf
```

### البنية التحتية للشبكة

ينشئ `network.tf` شبكة VNet والشبكة الفرعية وعنوان IP العام ومجموعة أمان الشبكة NSG (المنافذ 22/80/443/8888) وواجهة الشبكة NIC. المنفذ 8888 مطلوب لتطبيق crAPI الذي يعمل على منفذ مخصص:

```hcl file=../../terraform/network.tf
```

### الآلة الافتراضية مع Cloud-Init

ينشئ `vm.tf` آلة افتراضية Ubuntu 24.04 ويمرر سكربت تهيئة cloud-init. يوفر قرص SSD المتميز بسعة 60 جيبي بايت مساحة كافية لصور Docker ووحدات تخزين الحاويات:

```hcl file=../../terraform/vm.tf
```

### تهيئة Cloud-Init

يقوم `cloud-init.yaml` بتهيئة الآلة الافتراضية بتكوين محصّن. توفر مكتبة مساعدة مشتركة (`/usr/local/lib/cloud-init-helpers.sh`) منطق إعادة المحاولة لجميع عمليات الشبكة وتسجيل التقدم في `/var/log/cloud-init-progress.log`. يحل الاستقصاء النشط للتحقق من الصحة محل فترات الانتظار الثابتة لجاهزية الحاويات. يقوم بتهيئة:

- **ضبط النواة**: TCP keepalive، إعادة استخدام TIME_WAIT، 2 مليون tw_buckets، 64 ألف somaxconn
- **nginx**: 8 كتل upstream مع جلسات ثابتة (ip_hash, cookie hash)، مجمعات keepalive (64--128)، ذاكرة تخزين مؤقت للوكيل لـ Juice Shop، مستوى error_log crit
- **Docker Compose**: 41 حاوية عبر 9 تطبيقات مع حدود CPU/ذاكرة لكل حاوية
- **بناءات مخصصة**: DVWA-FPM (php-fpm مع pm.max_requests=200)، CSD Demo (Flask + gunicorn)، RESTaurant (مستنسخ من GitHub)
- **إعادة تدوير العمال**: uvicorn --limit-max-requests 200 (RESTaurant)، pm.max_requests 200 (DVWA-FPM) لمنع تسرب الذاكرة تحت الحمل المستمر

```yaml file=../../terraform/cloud-init.yaml
```

### المخرجات

يعرض `outputs.tf` عدد 25 مخرجًا وفقًا لمعيار الموارد التجريبية — 15 مخرجًا قياسيًا مشتركًا بين جميع الموارد التجريبية (`deployer`، `public_ip`، `private_ip`، `ssh_command`، `resource_group_name`، `vm_name`، `nsg_name`، `vnet_name`، `subnet_id`، `component`، `environment`، `resource_group_id`، `vm_id`، `nsg_id`، `location`) بالإضافة إلى 10 عناوين URL خاصة بتطبيقات المكونات (`origin_url`، `health_check_url`، `juice_shop_url`، `dvwa_url`، `vampi_url`، `httpbin_url`، `whoami_url`، `dvga_url`، `restaurant_url`، `crapi_url`):

```hcl file=../../terraform/outputs.tf
```

### ملف المتغيرات النموذجي

انسخ `terraform.tfvars.example` إلى `terraform.tfvars` واملأ القيم الخاصة بك. يستثني `.gitignore` ملف `terraform.tfvars` لمنع إرسال بيانات الاعتماد:

```hcl file=../../terraform/terraform.tfvars.example
```

## النشر

```bash
cp terraform.tfvars.example terraform.tfvars
# قم بتحرير terraform.tfvars بمعرف اشتراك Azure ومسار مفتاح SSH الخاص بك

terraform init

terraform plan

terraform apply
```

يعرض Terraform عنوان IP العام وأمر SSH وعناوين URL للتطبيقات بعد النشر الناجح.

## ما بعد النشر

بعد اكتمال `terraform apply`، انتظر من 5 إلى 10 دقائق حتى ينتهي cloud-init من تثبيت Docker وسحب صور الحاويات وتكوين nginx. عمليات سحب صور Docker هي العائق الرئيسي — صورة Juice Shop وحدها تبلغ حوالي 400 ميبي بايت.

تحقق من نقطة نهاية الصحة:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

الاستجابة المتوقعة:

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### إعداد قاعدة بيانات DVWA

يتطلب DVWA تهيئة قاعدة البيانات لمرة واحدة. افتح `http://<PUBLIC_IP>/dvwa/setup.php` في المتصفح وانقر على **Create / Reset Database**. بيانات الاعتماد الافتراضية هي `admin` / `password`.

:::note
يتم تهيئة قاعدة بيانات SQLite الخاصة بـ VAmPI تلقائيًا بواسطة سكربت نقطة دخول gunicorn. تقوم كل حاوية بتشغيل `db.create_all()` عند بدء التشغيل، لذا لا حاجة لأي تدخل يدوي.
:::

## الربط بالمكونات اللاحقة

بعد النشر، تستخدم المكونات الأخرى مخرجات خادم الأصل كمدخلات لها:

```bash
# محاكي CDN — مرر عنوان IP العام لخادم الأصل كمصدر upstream
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# مولد حركة المرور — مرر FQDN لموازن تحميل F5 XC (وليس عنوان IP الأصل مباشرة)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| المخرج | المكون اللاحق | متغير الإدخال |
|---|---|---|
| `public_ip` | محاكي CDN | `origin_server` (مع بادئة `http://`) |
| `public_ip` | محاكي CDN | `origin_host` (مع لاحقة `:80`، بدون مخطط) |
| `public_ip` | مجمع أصل F5 XC | عنوان خادم الأصل |

انتقل إلى [التطبيقات](../04-applications/) لأدلة الاستخدام أو [التحقق](../05-verify/) لاختبارات التحقق السريع.
