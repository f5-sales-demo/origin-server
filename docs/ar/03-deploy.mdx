جميع ملفات Terraform موجودة في مجلد [`terraform/`](https://github.com/f5xc-salesdemos/origin-server/tree/main/terraform). استنسخ المستودع وانشر مباشرةً:

```bash
git clone https://github.com/f5xc-salesdemos/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and SSH key path
```

## إعداد Terraform

### هيكل ملفات Terraform

يحتوي مجلد terraform على 9 ملفات تتبع معيار موارد العرض التوضيحي:

- `versions.tf` — قيود إصدارات Terraform والموفرين (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — إعداد موفري Azure RM وAzure AD
- `data.tf` — مصادر بيانات Azure AD للتحليل التلقائي للمنشئ
- `locals.tf` — تحليل المنشئ، وتسمية الموارد وفق إطار اعتماد Azure Cloud، والوسوم القياسية
- `main.tf` — مجموعة الموارد (المسماة `rg-origin-server-{environment}-{deployer}`)
- `variables.tf` — جميع متغيرات الإدخال (1 مطلوب، 8 اختيارية)
- `network.tf` — الشبكة الافتراضية (10.200.0.0/16)، والشبكة الفرعية، والعنوان IP العام، ومجموعة أمان الشبكة (المنافذ 22/80/443/8888)، وبطاقة واجهة الشبكة
- `vm.tf` — جهاز Ubuntu 24.04 الافتراضي مع cloud-init عبر templatefile()
- `outputs.tf` — 25 مخرجاً (15 قياسياً + 10 روابط URL خاصة بتطبيقات المكونات)

يحتوي `main.tf` على مجموعة الموارد فقط:

```hcl file=../_data/main.tf
```

يُعرِّف `variables.tf` جميع المعاملات القابلة للتهيئة. يتم تحليل معرّف `deployer` تلقائياً من حساب Azure AD الخاص بك — تحتاج فقط إلى ضبط `subscription_id`. القيمة الافتراضية لـ `vm_size` هي `Standard_D16s_v3` (16 vCPU، 64 جيجابايت RAM) مُحدَّدة لـ 41 حاوية Docker:

```hcl file=../_data/variables.tf
```

### البنية التحتية للشبكات

يُنشئ `network.tf` الشبكة الافتراضية والشبكة الفرعية والعنوان IP العام ومجموعة أمان الشبكة (المنافذ 22/80/443/8888) وبطاقة واجهة الشبكة. المنفذ 8888 مطلوب لـ crAPI الذي يعمل على منفذ مخصص:

```hcl file=../_data/network.tf
```

### الجهاز الافتراضي مع Cloud-Init

يُنشئ `vm.tf` جهاز Ubuntu 24.04 الافتراضي ويمرر سكريبت توفير cloud-init. يوفر القرص SSD Premium سعة 60 جيجابايت مساحةً كافيةً لصور Docker ووحدات تخزين الحاويات:

```hcl file=../_data/vm.tf
```

### توفير Cloud-Init

يُهيئ `cloud-init.yaml` الجهاز الافتراضي بإعداد مُحكم. توفر مكتبة مساعدة مشتركة (`/usr/local/lib/cloud-init-helpers.sh`) منطق إعادة المحاولة على جميع العمليات الشبكية وتسجيل التقدم في `/var/log/cloud-init-progress.log`. يستعيض استطلاع الفحص الصحي النشط عن وضع السكون ذي المدة الثابتة لجاهزية الحاوية. يُهيئ:

- **ضبط النواة**: TCP keepalive، وإعادة استخدام TIME_WAIT، و2M tw_buckets، و64K somaxconn
- **nginx**: 8 كتل upstream مع جلسات لاصقة (ip_hash، cookie hash)، ومجمعات keepalive (64--128)، وذاكرة وصول عشوائي (proxy cache) لـ Juice Shop، ومستوى error_log بدرجة crit
- **Docker Compose**: 41 حاوية عبر 9 تطبيقات مع حدود CPU/ذاكرة لكل حاوية
- **بنيات مخصصة**: DVWA-FPM (php-fpm مع pm.max_requests=200)، وCSD Demo (Flask + gunicorn)، وRESTaurant (مستنسخ من GitHub)
- **إعادة تدوير العمال**: uvicorn --limit-max-requests 200 (RESTaurant)، وpm.max_requests 200 (DVWA-FPM) لمنع تسرب الذاكرة تحت الحمل المستمر

```yaml file=../_data/cloud-init.yaml
```

### المخرجات

يكشف `outputs.tf` عن 25 مخرجاً وفق معيار موارد العرض التوضيحي — 15 مخرجاً قياسياً مشتركاً بين جميع موارد العرض التوضيحي (`deployer`، و`public_ip`، و`private_ip`، و`ssh_command`، و`resource_group_name`، و`vm_name`، و`nsg_name`، و`vnet_name`، و`subnet_id`، و`component`، و`environment`، و`resource_group_id`، و`vm_id`، و`nsg_id`، و`location`) بالإضافة إلى 10 روابط URL خاصة بتطبيقات المكونات (`origin_url`، و`health_check_url`، و`juice_shop_url`، و`dvwa_url`، و`vampi_url`، و`httpbin_url`، و`whoami_url`، و`dvga_url`، و`restaurant_url`، و`crapi_url`):

```hcl file=../_data/outputs.tf
```

### مثال على ملف المتغيرات

انسخ `terraform.tfvars.example` إلى `terraform.tfvars` واملأ قيمك. يستثني `.gitignore` ملف `terraform.tfvars` لمنع إيداع بيانات الاعتماد:

```hcl file=../_data/terraform.tfvars.example
```

## النشر

```bash
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and SSH key path

terraform init

terraform plan

terraform apply
```

تُخرج Terraform عنوان IP العام وأمر SSH وروابط URL للتطبيقات بعد النشر الناجح.

## ما بعد النشر

بعد اكتمال `terraform apply`، اسمح بمرور 5--10 دقائق حتى تنتهي cloud-init من تثبيت Docker وسحب صور الحاويات وضبط nginx. عمليات سحب صور Docker هي عنق الزجاجة -- إذ يبلغ حجم Juice Shop وحده نحو 400 ميجابايت.

تحقق من نقطة نهاية الفحص الصحي:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

الاستجابة المتوقعة:

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### إعداد قاعدة بيانات DVWA

يتطلب DVWA تهيئة قاعدة بيانات لمرة واحدة. افتح `http://<PUBLIC_IP>/dvwa/setup.php` في متصفح وانقر فوق **Create / Reset Database**. بيانات الاعتماد الافتراضية هي `admin` / `password`.

:::note
تتم تهيئة قاعدة بيانات SQLite الخاصة بـ VAmPI تلقائياً عبر سكريبت دخول gunicorn. تُشغِّل كل حاوية `db.create_all()` عند بدء التشغيل، لذا لا يلزم أي تدخل يدوي.
:::

## الربط بالمكونات التالية

بعد النشر، تستخدم المكونات الأخرى مخرجات خادم المصدر كمدخلات لها:

```bash
# CDN Simulator — pass origin-server's public IP as its upstream
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# Traffic Generator — pass the F5 XC load balancer FQDN (not the origin IP directly)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| المخرج | المكون التالي | متغير الإدخال |
|---|---|---|
| `public_ip` | محاكي CDN | `origin_server` (مع البادئة `http://`) |
| `public_ip` | محاكي CDN | `origin_host` (مع اللاحقة `:80`، بدون مخطط) |
| `public_ip` | مجموعة خوادم المصدر في F5 XC | عنوان خادم المصدر |

انتقل إلى [التطبيقات](../04-applications/) للاطلاع على أدلة الاستخدام أو [التحقق](../05-verify/) لاختبارات الدخان.
