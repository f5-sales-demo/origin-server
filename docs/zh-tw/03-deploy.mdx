所有 Terraform 檔案均位於 [`terraform/`](https://github.com/f5-sales-demo/origin-server/tree/main/terraform) 目錄中。克隆儲存庫並直接部署：

```bash
git clone https://github.com/f5-sales-demo/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and SSH key path
```

## Terraform 配置

### Terraform 檔案結構

terraform 目錄包含 9 個遵循展示資源標準的檔案：

- `versions.tf` — Terraform 及提供者版本約束（azurerm ~> 4.0、azuread ~> 3.0）
- `providers.tf` — Azure RM 與 Azure AD 提供者配置
- `data.tf` — 用於部署者自動解析的 Azure AD 資料來源
- `locals.tf` — 部署者解析、Azure 雲端採用框架資源命名、標準標籤
- `main.tf` — 資源群組（命名為 `rg-origin-server-{environment}-{deployer}`）
- `variables.tf` — 所有輸入變數（1 個必填，8 個選填）
- `network.tf` — VNet（10.200.0.0/16）、子網路、公用 IP、NSG（連接埠 22/80/443/8888）、NIC
- `vm.tf` — 透過 templatefile() 使用 cloud-init 的 Ubuntu 24.04 VM
- `outputs.tf` — 25 個輸出（15 個標準輸出 + 10 個元件專屬應用程式 URL）

`main.tf` 僅包含資源群組：

```hcl file=../_data/main.tf
```

`variables.tf` 定義所有可配置參數。`deployer` 識別碼會從您的 Azure AD 帳號自動解析——您只需設定 `subscription_id`。`vm_size` 預設值為 `Standard_D16s_v3`（16 vCPU、64 GiB RAM），已針對 41 個 Docker 容器調整大小：

```hcl file=../_data/variables.tf
```

### 網路基礎架構

`network.tf` 建立 VNet、子網路、公用 IP、NSG（連接埠 22/80/443/8888）及 NIC。連接埠 8888 是 crAPI 所必需的，因為它在專用連接埠上運行：

```hcl file=../_data/network.tf
```

### 搭配 Cloud-Init 的虛擬機器

`vm.tf` 建立 Ubuntu 24.04 VM 並傳遞 cloud-init 佈建腳本。60 GiB 進階 SSD 為 Docker 映像檔和容器磁碟區提供充足空間：

```hcl file=../_data/vm.tf
```

### Cloud-Init 佈建

`cloud-init.yaml` 以強化配置佈建 VM。共用輔助函式庫（`/usr/local/lib/cloud-init-helpers.sh`）為所有網路操作提供重試邏輯，並將進度日誌記錄至 `/var/log/cloud-init-progress.log`。主動健康檢查輪詢取代固定時間的等待，用於容器就緒確認。佈建內容包括：

- **核心調整**：TCP keepalive、TIME_WAIT 重用、2M tw_buckets、64K somaxconn
- **nginx**：8 個上游區塊，搭配黏性工作階段（ip_hash、cookie hash）、keepalive 連線池（64--128）、Juice Shop 的代理快取、錯誤日誌 crit 層級
- **Docker Compose**：9 個應用程式共 41 個容器，每個容器均有 CPU/記憶體限制
- **自訂建置**：DVWA-FPM（php-fpm 搭配 pm.max_requests=200）、CSD Demo（Flask + gunicorn）、RESTaurant（從 GitHub 克隆）
- **工作者回收**：uvicorn --limit-max-requests 200（RESTaurant）、pm.max_requests 200（DVWA-FPM），以防止持續負載下的記憶體洩漏

```yaml file=../_data/cloud-init.yaml
```

### 輸出

`outputs.tf` 遵循展示資源標準，公開 25 個輸出——15 個所有展示資源共用的標準輸出（`deployer`、`public_ip`、`private_ip`、`ssh_command`、`resource_group_name`、`vm_name`、`nsg_name`、`vnet_name`、`subnet_id`、`component`、`environment`、`resource_group_id`、`vm_id`、`nsg_id`、`location`），以及 10 個元件專屬應用程式 URL（`origin_url`、`health_check_url`、`juice_shop_url`、`dvwa_url`、`vampi_url`、`httpbin_url`、`whoami_url`、`dvga_url`、`restaurant_url`、`crapi_url`）：

```hcl file=../_data/outputs.tf
```

### 範例變數檔案

將 `terraform.tfvars.example` 複製為 `terraform.tfvars` 並填入您的值。`.gitignore` 已排除 `terraform.tfvars` 以防止提交憑證：

```hcl file=../_data/terraform.tfvars.example
```

## 部署

```bash
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and SSH key path

terraform init

terraform plan

terraform apply
```

Terraform 在成功部署後會輸出公用 IP、SSH 命令及應用程式 URL。

## 部署後續

`terraform apply` 完成後，請等待 5--10 分鐘，讓 cloud-init 完成 Docker 安裝、容器映像檔拉取及 nginx 配置。Docker 映像檔拉取是瓶頸所在——僅 Juice Shop 就約 400 MiB。

驗證健康端點：

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

預期回應：

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### DVWA 資料庫設定

DVWA 需要一次性的資料庫初始化。在瀏覽器中開啟 `http://<PUBLIC_IP>/dvwa/setup.php`，然後點擊 **Create / Reset Database**。預設憑證為 `admin` / `password`。

:::note
VAmPI 的 SQLite 資料庫由其 gunicorn 進入點腳本自動初始化。每個容器在啟動時執行 `db.create_all()`，因此無需手動介入。
:::

## 與下游元件對接

部署完成後，其他元件將使用來源伺服器的輸出作為其輸入：

```bash
# CDN Simulator — pass origin-server's public IP as its upstream
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# Traffic Generator — pass the F5 XC load balancer FQDN (not the origin IP directly)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| 輸出 | 下游元件 | 輸入變數 |
|---|---|---|
| `public_ip` | CDN Simulator | `origin_server`（加上 `http://` 前綴） |
| `public_ip` | CDN Simulator | `origin_host`（加上 `:80` 後綴，不含協議） |
| `public_ip` | F5 XC Origin Pool | 來源伺服器位址 |
