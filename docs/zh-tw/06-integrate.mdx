## F5 XC 來源集區整合

此來源伺服器設計為在 F5 Distributed Cloud HTTP 負載平衡器後方作為**來源集區成員**使用。F5 XC 平台在此來源前方提供 WAF、Bot Defense、API Security 和 Client-Side Defense。

### 架構

```
終端使用者 -> F5 XC HTTP LB (WAF/Bot/API/CSD) -> 來源伺服器虛擬機器 (nginx -> Docker 應用程式)
```

### 來源集區配置

建立指向此虛擬機器公用 IP 的 F5 XC 來源集區：

| 設定 | 值 |
|---|---|
| **來源伺服器類型** | 來源伺服器的公用 IP |
| **IP 位址** | `<terraform output public_ip>` |
| **連接埠** | `80` |
| **健康檢查** | HTTP，路徑 `/health` |

### HTTP 負載平衡器配置

使用來源集區建立 HTTP 負載平衡器：

| 設定 | 值 |
|---|---|
| **網域** | 您的示範網域（例如 `demo.example.com`） |
| **來源集區** | 上方建立的集區 |
| **WAF** | 附加您的 WAF 策略 |
| **Bot Defense** | 根據需要啟用 |
| **API Discovery** | 為 VAmPI 端點啟用 |

### 基於路徑的路由

每個應用程式都可透過負載平衡器的路徑前綴存取：

| F5 XC LB URL | 來源路徑 | 應用程式 |
|---|---|---|
| `https://demo.example.com/juice-shop/` | `/juice-shop/` | Juice Shop（4 個實例，cookie 黏著） |
| `https://demo.example.com/dvwa/` | `/dvwa/` | DVWA（4 個實例 + MariaDB，cookie 黏著） |
| `https://demo.example.com/vampi/` | `/vampi/` | VAmPI（4 個實例，ip_hash 黏著） |
| `https://demo.example.com/httpbin/` | `/httpbin/` | httpbin（4 個實例，輪詢） |
| `https://demo.example.com/whoami/` | `/whoami/` | 請求診斷（4 個實例） |
| `https://demo.example.com/csd-demo/` | `/csd-demo/` | CSD Demo（4 個實例，ip_hash 黏著） |
| `https://demo.example.com/dvga/` | `/dvga/` | DVGA（4 個實例，ip_hash 黏著） |
| `https://demo.example.com/restaurant/` | `/restaurant/` | RESTaurant（4 個實例，輪詢） |
| `https://demo.example.com/health` | `/health` | 健康檢查（nginx 直接處理） |

### 驗證 F5 XC 標頭注入

使用 whoami 端點驗證 F5 XC 注入到達來源請求的標頭：

```bash
LB_DOMAIN="demo.example.com"

curl -sk "https://${LB_DOMAIN}/whoami/"
```

在回應中尋找這些 F5 XC 注入的標頭：

| 標頭 | 意義 |
|---|---|
| `X-Forwarded-For` | 透過 F5 XC 的用戶端 IP 鏈 |
| `True-Client-IP` | 原始用戶端 IP |
| `X-Forwarded-Proto` | 如果 TLS 在 F5 XC 終止則為 `https` |
| `X-Volterra-Bot-Type` | Bot 分類（啟用 Bot Defense 時） |
| `X-Request-ID` | F5 XC 請求追蹤 ID |

### 透過 F5 XC 進行 WAF 測試

將 WAF 策略附加到 HTTP 負載平衡器後，測試攻擊酬載：

```bash
LB_DOMAIN="demo.example.com"

# SQL 注入（應被 WAF 阻擋）
curl -sk "https://${LB_DOMAIN}/dvwa/vulnerabilities/sqli/?id=%27+OR+1%3D1--&Submit=Submit"

# XSS（應被 WAF 阻擋）
curl -sk "https://${LB_DOMAIN}/juice-shop/rest/products/search?q=<script>alert(1)</script>"

# 命令注入（應被 WAF 阻擋）
curl -sk "https://${LB_DOMAIN}/dvwa/vulnerabilities/exec/" \
  -d "ip=127.0.0.1;cat+/etc/passwd&Submit=Submit"

# 正常請求（應通過）
curl -sk "https://${LB_DOMAIN}/httpbin/get" | jq .
```

### API 安全測試

在 HTTP 負載平衡器上啟用 API Discovery 以映射 VAmPI 的 API 端點：

```bash
# 這些請求在 F5 XC 中建立 API 清單
curl -sk "https://${LB_DOMAIN}/vampi/users/v1"
curl -sk -X POST "https://${LB_DOMAIN}/vampi/users/v1/register" \
  -H "Content-Type: application/json" \
  -d '{"username":"apitest","password":"test123","email":"api@test.com"}'
curl -sk -X POST "https://${LB_DOMAIN}/vampi/users/v1/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"apitest","password":"test123"}'
```

### crAPI 整合（連接埠 8888）

crAPI 在專用連接埠（8888）上執行，因為它是一個單頁應用程式，其 API 路徑是硬編碼的，無法在路徑前綴後方提供服務。要將 crAPI 與 F5 XC 整合：

| 設定 | 值 |
|---|---|
| **來源伺服器類型** | 來源伺服器的公用 IP |
| **IP 位址** | `<terraform output public_ip>` |
| **連接埠** | `8888` |
| **健康檢查** | HTTP，路徑 `/` |

為連接埠 8888 上的 crAPI 建立**獨立的來源集區**，或將第二個來源集區成員（使用連接埠 8888）新增到現有集區，並使用路由規則導向流量：

```bash
# 透過 F5 XC 測試 crAPI（如果已配置）
curl -sk "https://${LB_DOMAIN}:8888/"

# 或者如果在同一 LB 網域上使用路由規則：
# 配置 F5 XC 路由規則，比對 Host 標頭或路徑前綴
# 以轉發到 crAPI 來源集區（連接埠 8888）
```

:::note
來源虛擬機器上的 NSG 允許連接埠 8888 的入站流量。無需額外的 Azure 網路變更。
:::

### 黏著工作階段注意事項

來源伺服器內部使用 nginx 黏著工作階段，將有狀態應用程式路由到一致的後端容器。配置 F5 XC HTTP 負載平衡器時，請注意：

| 應用程式 | 黏著方法 | 原因 |
|---|---|---|
| Juice Shop | `hash $cookie_token` | Node.js 工作階段狀態 |
| DVWA | `hash $cookie_PHPSESSID` | PHP 工作階段狀態 |
| VAmPI | `ip_hash` | 每個實例的 SQLite 資料庫 |
| CSD Demo | `ip_hash` | 每個實例的記憶體內洩漏日誌 |
| DVGA | `ip_hash` | 每個實例的 SQLite 資料庫 |
| RESTaurant | 輪詢 | 共享 PostgreSQL 後端 |
| crAPI | --（單一連接埠 8888） | 7 個微服務，PostgreSQL + MongoDB |
| httpbin | 輪詢 | 無狀態 |
| whoami | 輪詢 | 無狀態 |

F5 XC 不需要複製這些黏著工作階段 -- 來源虛擬機器上的 nginx 處理後端路由。F5 XC 應將來源視為單一端點（虛擬機器在連接埠 80 的公用 IP）。

### 多組件架構

與 CDN 模擬器結合時，完整的實驗室架構為：

```
終端使用者 -> CDN 模擬器（nginx 快取） -> F5 XC HTTP LB -> 來源伺服器（此組件）
```

CDN 模擬器使用 F5 XC HTTP 負載平衡器 VIP 作為其來源，而 F5 XC 負載平衡器使用此來源伺服器作為其後端。這創建了一個真實的多層架構，用於全面測試。
