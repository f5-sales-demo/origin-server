## आवश्यक

### Azure सब्सक्रिप्शन

निम्नलिखित बनाने की अनुमति के साथ एक सक्रिय Azure सब्सक्रिप्शन:

- रिसोर्स ग्रुप
- वर्चुअल नेटवर्क और सबनेट
- नेटवर्क सिक्योरिटी ग्रुप
- पब्लिक IP एड्रेस
- वर्चुअल मशीन (Standard_D16s_v3 -- 16 vCPU, 64 GiB RAM, 41-कंटेनर वर्कलोड के लिए)

### Azure CLI

Azure CLI इंस्टॉल और प्रमाणित करें:

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

### Terraform

Terraform >= 1.5 आवश्यक है:

```bash
terraform version
```

यदि इंस्टॉल नहीं है, तो HashiCorp APT रिपॉजिटरी का उपयोग करें:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### Azure AD अनुमतियाँ

आपके Azure AD अकाउंट में `User.Read` अनुमतियाँ होनी चाहिए। डिप्लॉयर आइडेंटिफायर (जो रिसोर्स नामकरण में उपयोग किया जाता है) स्वचालित रूप से आपकी Azure AD प्रोफ़ाइल से प्राप्त किया जाता है। सर्विस प्रिंसिपल या मैनेज्ड आइडेंटिटी प्रमाणीकरण के लिए, `deployer` Terraform वेरिएबल को स्पष्ट रूप से सेट करें।

### SSH कुंजी जोड़ी

VM एक्सेस के लिए एक SSH कुंजी जोड़ी:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/origin-server-key -N ""
```

## निर्मित Azure रिसोर्स

| रिसोर्स | SKU |
|---|---|
| Ubuntu 24.04 VM | Standard_D16s_v3 (16 vCPU, 64 GiB) |
| पब्लिक IP | Standard, Static |
| OS डिस्क | 60 GiB Premium SSD |
| VNet + NSG | Default |

वर्तमान लागतों के लिए [Azure मूल्य कैलकुलेटर](https://azure.microsoft.com/en-us/pricing/calculator/) देखें। जब लैब उपयोग में न हो तो शुल्क रोकने के लिए `terraform destroy` का उपयोग करें। प्रक्रिया के लिए [टियरडाउन](../07-teardown/) देखें।

:::note
41-कंटेनर डिप्लॉयमेंट के लिए Standard_D16s_v3 (64 GiB RAM) आवश्यक है। कंटेनर मेमोरी आवंटन: 4 Juice Shop इंस्टेंस प्रत्येक 1 GiB (4 GiB), 4 VAmPI प्रत्येक 512 MiB (2 GiB), 4 DVWA-FPM + MariaDB (1.8 GiB), 4 httpbin प्रत्येक 256 MiB (1 GiB), 4 CSD Demo प्रत्येक 256 MiB (1 GiB), 4 whoami प्रत्येक 64 MiB (256 MiB), 4 DVGA प्रत्येक 256 MiB (1 GiB), 4 RESTaurant + PostgreSQL (1.5 GiB), crAPI 7 माइक्रोसर्विसेज (2 GiB), साथ ही nginx, OS, और बफर कैश। कुल कंटेनर आवंटन लगभग 14.5 GiB (64 GiB का 22.7%) है।
:::

:::note
NSG पोर्ट 80 (HTTP) और 8888 (crAPI) पर इनबाउंड ट्रैफिक की अनुमति देता है। पोर्ट 8888 आवश्यक है क्योंकि crAPI एक सिंगल-पेज एप्लिकेशन है जो अपने API पथों को हार्डकोड करता है और पाथ प्रीफिक्स के पीछे सर्व नहीं किया जा सकता।
:::
