सभी Terraform फाइलें [`terraform/`](https://github.com/f5xc-salesdemos/origin-server/tree/main/terraform) डायरेक्टरी में हैं। रिपॉज़िटरी क्लोन करें और सीधे डिप्लॉय करें:

```bash
git clone https://github.com/f5xc-salesdemos/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# terraform.tfvars को अपने Azure subscription ID और SSH key path के साथ संपादित करें
```

## Terraform कॉन्फ़िगरेशन

### Terraform फाइल संरचना

terraform डायरेक्टरी में डेमो रिसोर्स स्टैंडर्ड के अनुसार 9 फाइलें हैं:

- `versions.tf` — Terraform और प्रोवाइडर वर्शन कंस्ट्रेंट्स (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — Azure RM और Azure AD प्रोवाइडर कॉन्फ़िगरेशन
- `data.tf` — डिप्लॉयर ऑटो-रिज़ॉल्यूशन के लिए Azure AD डेटा सोर्स
- `locals.tf` — डिप्लॉयर रिज़ॉल्यूशन, Azure Cloud Adoption Framework रिसोर्स नेमिंग, स्टैंडर्ड टैग
- `main.tf` — रिसोर्स ग्रुप (नाम `rg-origin-server-{environment}-{deployer}`)
- `variables.tf` — सभी इनपुट वेरिएबल (1 आवश्यक, 8 वैकल्पिक)
- `network.tf` — VNet (10.200.0.0/16), सबनेट, पब्लिक IP, NSG (पोर्ट 22/80/443/8888), NIC
- `vm.tf` — templatefile() के माध्यम से cloud-init के साथ Ubuntu 24.04 VM
- `outputs.tf` — 25 आउटपुट (15 स्टैंडर्ड + 10 कंपोनेंट-विशिष्ट एप्लिकेशन URL)

`main.tf` में केवल रिसोर्स ग्रुप है:

```hcl file=./_data/main.tf
```

`variables.tf` सभी कॉन्फ़िगर करने योग्य पैरामीटर परिभाषित करता है। `deployer` आइडेंटिफायर आपके Azure AD अकाउंट से स्वचालित रूप से रिज़ॉल्व होता है — आपको केवल `subscription_id` सेट करना होगा। `vm_size` का डिफ़ॉल्ट `Standard_D16s_v3` (16 vCPU, 64 GiB RAM) है जो 41 Docker कंटेनरों के लिए उपयुक्त है:

```hcl file=./_data/variables.tf
```

`network.tf` VNet, सबनेट, पब्लिक IP, NSG (पोर्ट 22/80/443/8888), और NIC बनाता है। पोर्ट 8888 crAPI के लिए आवश्यक है जो एक समर्पित पोर्ट पर चलता है:

### Cloud-Init के साथ वर्चुअल मशीन

`vm.tf` Ubuntu 24.04 VM बनाता है और cloud-init प्रोविज़निंग स्क्रिप्ट पास करता है। 60 GiB Premium SSD Docker इमेज और कंटेनर वॉल्यूम के लिए पर्याप्त स्थान प्रदान करता है:

```hcl file=./_data/vm.tf
```

### Cloud-Init प्रोविज़निंग

`cloud-init.yaml` VM को हार्डेंड कॉन्फ़िगरेशन के साथ प्रोविज़न करता है। एक शेयर्ड हेल्पर लाइब्रेरी (`/usr/local/lib/cloud-init-helpers.sh`) सभी नेटवर्क ऑपरेशनों पर रिट्राई लॉजिक और `/var/log/cloud-init-progress.log` में प्रगति लॉगिंग प्रदान करती है। कंटेनर रेडीनेस के लिए फिक्स्ड-ड्यूरेशन स्लीप की जगह एक्टिव हेल्थ-चेक पोलिंग का उपयोग होता है। यह प्रोविज़न करता है:

- **कर्नेल ट्यूनिंग**: TCP keepalive, TIME_WAIT reuse, 2M tw_buckets, 64K somaxconn
- **nginx**: स्टिकी सेशन (ip_hash, cookie hash) के साथ 8 अपस्ट्रीम ब्लॉक, keepalive पूल (64--128), Juice Shop के लिए प्रॉक्सी कैश, error_log crit लेवल
- **Docker Compose**: प्रति-कंटेनर CPU/मेमोरी लिमिट के साथ 9 एप्लिकेशन में 41 कंटेनर
- **कस्टम बिल्ड**: DVWA-FPM (pm.max_requests=200 के साथ php-fpm), CSD Demo (Flask + gunicorn), RESTaurant (GitHub से क्लोन)
- **वर्कर रीसाइक्लिंग**: सतत लोड के तहत मेमोरी लीक रोकने के लिए uvicorn --limit-max-requests 200 (RESTaurant), pm.max_requests 200 (DVWA-FPM)

```yaml file=./_data/cloud-init.yaml
```

## डिप्लॉय

```bash
cp terraform.tfvars.example terraform.tfvars
# terraform.tfvars को अपने Azure subscription ID और SSH key path के साथ संपादित करें

terraform init

terraform plan

terraform apply
```

सफल डिप्लॉयमेंट के बाद Terraform पब्लिक IP, SSH कमांड, और एप्लिकेशन URL आउटपुट करता है।

## डिप्लॉय के बाद

`terraform apply` पूरा होने के बाद, cloud-init को Docker इंस्टॉल करने, कंटेनर इमेज पुल करने, और nginx कॉन्फ़िगर करने के लिए 5--10 मिनट का समय दें। Docker इमेज पुल बाधक हैं -- अकेले Juice Shop ~400 MiB का है।

हेल्थ एंडपॉइंट सत्यापित करें:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

अपेक्षित प्रतिक्रिया:

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### DVWA डेटाबेस सेटअप

DVWA को एक बार डेटाबेस इनिशियलाइज़ेशन की आवश्यकता होती है। ब्राउज़र में `http://<PUBLIC_IP>/dvwa/setup.php` खोलें और **Create / Reset Database** पर क्लिक करें। डिफ़ॉल्ट क्रेडेंशियल `admin` / `password` हैं।

:::note
VAmPI का SQLite डेटाबेस इसकी gunicorn एंट्रीपॉइंट स्क्रिप्ट द्वारा स्वचालित रूप से इनिशियलाइज़ होता है। प्रत्येक कंटेनर स्टार्टअप पर `db.create_all()` चलाता है, इसलिए किसी मैनुअल हस्तक्षेप की आवश्यकता नहीं है।
:::

## डाउनस्ट्रीम कंपोनेंट से कनेक्ट करना

डिप्लॉयमेंट के बाद, अन्य कंपोनेंट ऑरिजिन सर्वर के आउटपुट को अपने इनपुट के रूप में उपयोग करते हैं:

```bash
# CDN Simulator — origin-server का पब्लिक IP इसके अपस्ट्रीम के रूप में पास करें
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# Traffic Generator — F5 XC लोड बैलेंसर FQDN पास करें (सीधे ऑरिजिन IP नहीं)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| आउटपुट | डाउनस्ट्रीम कंपोनेंट | इनपुट वेरिएबल |
|---|---|---|
| `public_ip` | CDN Simulator | `origin_server` (`http://` प्रीफिक्स के साथ) |
| `public_ip` | CDN Simulator | `origin_host` (`:80` सफिक्स के साथ, बिना स्कीम) |
| `public_ip` | F5 XC Origin Pool | ऑरिजिन सर्वर एड्रेस |

उपयोग गाइड के लिए [एप्लिकेशन](../04-applications/) या स्मोक टेस्ट के लिए [सत्यापन](../05-verify/) पर आगे बढ़ें।
