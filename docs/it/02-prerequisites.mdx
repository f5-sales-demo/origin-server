## Requisiti

### Sottoscrizione Azure

Una sottoscrizione Azure attiva con i permessi per creare:

- Gruppi di risorse
- Reti virtuali e subnet
- Gruppi di sicurezza di rete
- Indirizzi IP pubblici
- Macchine virtuali (Standard_D16s_v3 -- 16 vCPU, 64 GiB RAM per carichi di lavoro a 41 container)

### Azure CLI

Installare e autenticarsi con Azure CLI:

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

### Terraform

È richiesto Terraform >= 1.5:

```bash
terraform version
```

Se non installato, utilizzare il repository APT di HashiCorp:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### Permessi Azure AD

Il vostro account Azure AD deve disporre dei permessi `User.Read`. L'identificativo del deployer (utilizzato nella denominazione delle risorse) viene derivato automaticamente dal vostro profilo Azure AD. Per l'autenticazione tramite service principal o managed identity, impostare esplicitamente la variabile Terraform `deployer`.

### Coppia di chiavi SSH

Una coppia di chiavi SSH per l'accesso alla VM:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/origin-server-key -N ""
```

## Risorse Azure create

| Risorsa | SKU |
|---|---|
| VM Ubuntu 24.04 | Standard_D16s_v3 (16 vCPU, 64 GiB) |
| IP pubblico | Standard, Statico |
| Disco OS | 60 GiB Premium SSD |
| VNet + NSG | Default |

Consultare il [calcolatore prezzi Azure](https://azure.microsoft.com/en-us/pricing/calculator/) per i costi aggiornati. Utilizzare `terraform destroy` quando il laboratorio non è in uso per interrompere gli addebiti. Vedere [Smantellamento](../07-teardown/) per la procedura.

:::note
Standard_D16s_v3 (64 GiB RAM) è richiesto per il deployment di 41 container. Allocazione memoria dei container: 4 istanze Juice Shop a 1 GiB ciascuna (4 GiB), 4 VAmPI a 512 MiB ciascuna (2 GiB), 4 DVWA-FPM + MariaDB (1,8 GiB), 4 httpbin a 256 MiB (1 GiB), 4 CSD Demo a 256 MiB (1 GiB), 4 whoami a 64 MiB (256 MiB), 4 DVGA a 256 MiB ciascuna (1 GiB), 4 RESTaurant + PostgreSQL (1,5 GiB), crAPI 7 microservizi (2 GiB), più nginx, OS e buffer cache. L'allocazione totale dei container è di circa 14,5 GiB (22,7% di 64 GiB).
:::

:::note
Il NSG consente il traffico in ingresso sulle porte 80 (HTTP) e 8888 (crAPI). La porta 8888 è necessaria perché crAPI è un'applicazione a pagina singola che ha i percorsi API codificati staticamente e non può essere servita dietro un prefisso di percorso.
:::
