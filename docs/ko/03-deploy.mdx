모든 Terraform 파일은 [`terraform/`](https://github.com/f5xc-salesdemos/origin-server/tree/main/terraform) 디렉토리에 있습니다. 리포지토리를 클론하고 바로 배포하세요:

```bash
git clone https://github.com/f5xc-salesdemos/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# terraform.tfvars에 Azure 구독 ID와 SSH 키 경로를 입력하세요
```

## Terraform 구성

### Terraform 파일 구조

terraform 디렉토리에는 데모 리소스 표준을 따르는 9개의 파일이 포함되어 있습니다:

- `versions.tf` — Terraform 및 프로바이더 버전 제약 조건 (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — Azure RM 및 Azure AD 프로바이더 구성
- `data.tf` — 배포자 자동 해석을 위한 Azure AD 데이터 소스
- `locals.tf` — 배포자 해석, Azure 클라우드 채택 프레임워크 리소스 명명, 표준 태그
- `main.tf` — 리소스 그룹 (이름: `rg-origin-server-{environment}-{deployer}`)
- `variables.tf` — 모든 입력 변수 (필수 1개, 선택 8개)
- `network.tf` — VNet (10.200.0.0/16), 서브넷, 공용 IP, NSG (포트 22/80/443/8888), NIC
- `vm.tf` — templatefile()을 통한 cloud-init이 포함된 Ubuntu 24.04 VM
- `outputs.tf` — 25개 출력 (표준 15개 + 컴포넌트별 애플리케이션 URL 10개)

`main.tf`에는 리소스 그룹만 포함되어 있습니다:

```hcl file=./_data/main.tf
```

`variables.tf`는 모든 구성 가능한 매개변수를 정의합니다. `deployer` 식별자는 Azure AD 계정에서 자동으로 해석됩니다 — `subscription_id`만 설정하면 됩니다. `vm_size` 기본값은 `Standard_D16s_v3` (16 vCPU, 64 GiB RAM)으로 41개의 Docker 컨테이너에 적합한 크기입니다:

```hcl file=./_data/variables.tf
```

### 네트워크 인프라

`network.tf`는 VNet, 서브넷, 공용 IP, NSG (포트 22/80/443/8888), NIC를 생성합니다. 포트 8888은 전용 포트에서 실행되는 crAPI에 필요합니다:

```hcl file=./_data/network.tf
```

### Cloud-Init을 사용한 가상 머신

`vm.tf`는 Ubuntu 24.04 VM을 생성하고 cloud-init 프로비저닝 스크립트를 전달합니다. 60 GiB Premium SSD는 Docker 이미지와 컨테이너 볼륨을 위한 충분한 공간을 제공합니다:

```hcl file=./_data/vm.tf
```

### Cloud-Init 프로비저닝

`cloud-init.yaml`은 강화된 구성으로 VM을 프로비저닝합니다. 공유 헬퍼 라이브러리 (`/usr/local/lib/cloud-init-helpers.sh`)는 모든 네트워크 작업에 대한 재시도 로직과 `/var/log/cloud-init-progress.log`에 대한 진행 상태 로깅을 제공합니다. 컨테이너 준비 상태를 위해 고정 기간 sleep 대신 능동적 헬스 체크 폴링을 사용합니다. 프로비저닝 내용:

- **커널 튜닝**: TCP keepalive, TIME_WAIT 재사용, 2M tw_buckets, 64K somaxconn
- **nginx**: sticky 세션 (ip_hash, cookie hash)이 포함된 8개 업스트림 블록, keepalive 풀 (64--128), Juice Shop용 프록시 캐시, error_log crit 레벨
- **Docker Compose**: 컨테이너별 CPU/메모리 제한이 있는 9개 애플리케이션에 걸친 41개 컨테이너
- **커스텀 빌드**: DVWA-FPM (pm.max_requests=200인 php-fpm), CSD Demo (Flask + gunicorn), RESTaurant (GitHub에서 클론)
- **워커 리사이클링**: 지속적인 부하 하에서 메모리 누수를 방지하기 위한 uvicorn --limit-max-requests 200 (RESTaurant), pm.max_requests 200 (DVWA-FPM)

```yaml file=./_data/cloud-init.yaml
```

### 출력

`outputs.tf`는 데모 리소스 표준을 따르는 25개의 출력을 노출합니다 — 모든 데모 리소스가 공유하는 15개의 표준 출력 (`deployer`, `public_ip`, `private_ip`, `ssh_command`, `resource_group_name`, `vm_name`, `nsg_name`, `vnet_name`, `subnet_id`, `component`, `environment`, `resource_group_id`, `vm_id`, `nsg_id`, `location`) 및 10개의 컴포넌트별 애플리케이션 URL (`origin_url`, `health_check_url`, `juice_shop_url`, `dvwa_url`, `vampi_url`, `httpbin_url`, `whoami_url`, `dvga_url`, `restaurant_url`, `crapi_url`):

```hcl file=./_data/outputs.tf
```

### 변수 파일 예시

`terraform.tfvars.example`을 `terraform.tfvars`로 복사하고 값을 입력하세요. `.gitignore`는 자격 증명 커밋을 방지하기 위해 `terraform.tfvars`를 제외합니다:

```hcl file=./_data/terraform.tfvars.example
```

## 배포

```bash
cp terraform.tfvars.example terraform.tfvars
# terraform.tfvars에 Azure 구독 ID와 SSH 키 경로를 입력하세요

terraform init

terraform plan

terraform apply
```

배포가 성공적으로 완료되면 Terraform이 공용 IP, SSH 명령어, 애플리케이션 URL을 출력합니다.

## 배포 후 작업

`terraform apply`가 완료된 후 cloud-init이 Docker 설치, 컨테이너 이미지 풀링, nginx 구성을 완료할 때까지 5--10분 정도 기다려 주세요. Docker 이미지 풀이 병목 구간입니다 -- Juice Shop만 해도 ~400 MiB입니다.

헬스 엔드포인트를 확인하세요:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

예상 응답:

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### DVWA 데이터베이스 설정

DVWA는 일회성 데이터베이스 초기화가 필요합니다. 브라우저에서 `http://<PUBLIC_IP>/dvwa/setup.php`를 열고 **Create / Reset Database**를 클릭하세요. 기본 자격 증명은 `admin` / `password`입니다.

:::note
VAmPI의 SQLite 데이터베이스는 gunicorn 엔트리포인트 스크립트에 의해 자동으로 초기화됩니다. 각 컨테이너는 시작 시 `db.create_all()`을 실행하므로 수동 개입이 필요하지 않습니다.
:::

## 다운스트림 컴포넌트 연결

배포 후 다른 컴포넌트는 오리진 서버의 출력을 입력으로 사용합니다:

```bash
# CDN Simulator — origin-server의 공용 IP를 업스트림으로 전달
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# Traffic Generator — F5 XC 로드 밸런서 FQDN을 전달 (오리진 IP를 직접 전달하지 않음)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| 출력 | 다운스트림 컴포넌트 | 입력 변수 |
|---|---|---|
| `public_ip` | CDN Simulator | `origin_server` (`http://` 접두사 포함) |
| `public_ip` | CDN Simulator | `origin_host` (`:80` 접미사 포함, 스킴 없음) |
| `public_ip` | F5 XC Origin Pool | 오리진 서버 주소 |

사용 가이드는 [애플리케이션](../04-applications/)으로, 스모크 테스트는 [검증](../05-verify/)으로 진행하세요.
