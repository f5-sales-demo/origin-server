ไฟล์ Terraform ทั้งหมดอยู่ในไดเรกทอรี [`terraform/`](https://github.com/f5xc-salesdemos/origin-server/tree/main/terraform) โคลน repository และ deploy ได้โดยตรง:

```bash
git clone https://github.com/f5xc-salesdemos/origin-server.git
cd origin-server/terraform
cp terraform.tfvars.example terraform.tfvars
# แก้ไข terraform.tfvars ด้วย Azure subscription ID และ SSH key path ของคุณ
```

## การกำหนดค่า Terraform

### โครงสร้างไฟล์ Terraform

ไดเรกทอรี terraform ประกอบด้วย 9 ไฟล์ตามมาตรฐาน Demo Resource Standard:

- `versions.tf` — ข้อจำกัดเวอร์ชัน Terraform และ provider (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` — การกำหนดค่า Azure RM และ Azure AD provider
- `data.tf` — Azure AD data sources สำหรับการ auto-resolution ของ deployer
- `locals.tf` — การ resolve deployer, การตั้งชื่อทรัพยากรตาม Azure Cloud Adoption Framework, แท็กมาตรฐาน
- `main.tf` — Resource group (ชื่อ `rg-origin-server-{environment}-{deployer}`)
- `variables.tf` — ตัวแปร input ทั้งหมด (1 จำเป็น, 8 ไม่บังคับ)
- `network.tf` — VNet (10.200.0.0/16), subnet, public IP, NSG (พอร์ต 22/80/443/8888), NIC
- `vm.tf` — VM Ubuntu 24.04 พร้อม cloud-init ผ่าน templatefile()
- `outputs.tf` — 25 outputs (15 มาตรฐาน + 10 URL แอปพลิเคชันเฉพาะคอมโพเนนต์)

`main.tf` มีเฉพาะ resource group เท่านั้น:

```hcl file=../../terraform/main.tf
```

`variables.tf` กำหนดพารามิเตอร์ที่ปรับแต่งได้ทั้งหมด ตัวระบุ `deployer` จะถูก auto-resolve จากบัญชี Azure AD ของคุณ — คุณต้องตั้งค่าเพียง `subscription_id` เท่านั้น ค่าเริ่มต้นของ `vm_size` คือ `Standard_D16s_v3` (16 vCPU, 64 GiB RAM) ซึ่งมีขนาดเพียงพอสำหรับ 41 Docker containers:

```hcl file=../../terraform/variables.tf
```

### โครงสร้างพื้นฐานเครือข่าย

`network.tf` สร้าง VNet, subnet, public IP, NSG (พอร์ต 22/80/443/8888) และ NIC พอร์ต 8888 จำเป็นสำหรับ crAPI ซึ่งทำงานบนพอร์ตเฉพาะ:

```hcl file=../../terraform/network.tf
```

### Virtual Machine พร้อม Cloud-Init

`vm.tf` สร้าง VM Ubuntu 24.04 และส่งสคริปต์ cloud-init provisioning Premium SSD ขนาด 60 GiB มีพื้นที่เพียงพอสำหรับ Docker images และ container volumes:

```hcl file=../../terraform/vm.tf
```

### Cloud-Init Provisioning

`cloud-init.yaml` จัดเตรียม VM ด้วยการกำหนดค่าที่ผ่านการ harden ไลบรารี helper ที่ใช้ร่วมกัน (`/usr/local/lib/cloud-init-helpers.sh`) มี retry logic สำหรับการดำเนินการเครือข่ายทั้งหมดและบันทึกความคืบหน้าไปยัง `/var/log/cloud-init-progress.log` การตรวจสอบ health-check แบบ active polling ถูกใช้แทน sleep ที่มีระยะเวลาคงที่สำหรับความพร้อมของ container โดยจัดเตรียม:

- **การปรับแต่ง Kernel**: TCP keepalive, การใช้ซ้ำ TIME_WAIT, tw_buckets 2M, somaxconn 64K
- **nginx**: 8 upstream blocks พร้อม sticky sessions (ip_hash, cookie hash), keepalive pools (64--128), proxy cache สำหรับ Juice Shop, error_log ระดับ crit
- **Docker Compose**: 41 containers ครอบคลุม 9 แอปพลิเคชันพร้อมขีดจำกัด CPU/memory ต่อ container
- **Custom builds**: DVWA-FPM (php-fpm พร้อม pm.max_requests=200), CSD Demo (Flask + gunicorn), RESTaurant (โคลนจาก GitHub)
- **การรีไซเคิล Worker**: uvicorn --limit-max-requests 200 (RESTaurant), pm.max_requests 200 (DVWA-FPM) เพื่อป้องกัน memory leaks ภายใต้โหลดที่ต่อเนื่อง

```yaml file=../../terraform/cloud-init.yaml
```

### Outputs

`outputs.tf` แสดง 25 outputs ตามมาตรฐาน Demo Resource Standard — 15 outputs มาตรฐานที่ใช้ร่วมกันโดย demo resources ทั้งหมด (`deployer`, `public_ip`, `private_ip`, `ssh_command`, `resource_group_name`, `vm_name`, `nsg_name`, `vnet_name`, `subnet_id`, `component`, `environment`, `resource_group_id`, `vm_id`, `nsg_id`, `location`) บวกกับ 10 URL แอปพลิเคชันเฉพาะคอมโพเนนต์ (`origin_url`, `health_check_url`, `juice_shop_url`, `dvwa_url`, `vampi_url`, `httpbin_url`, `whoami_url`, `dvga_url`, `restaurant_url`, `crapi_url`):

```hcl file=../../terraform/outputs.tf
```

### ตัวอย่างไฟล์ตัวแปร

คัดลอก `terraform.tfvars.example` ไปเป็น `terraform.tfvars` และกรอกค่าของคุณ `.gitignore` จะไม่รวม `terraform.tfvars` เพื่อป้องกันการ commit ข้อมูลรับรอง:

```hcl file=../../terraform/terraform.tfvars.example
```

## Deploy

```bash
cp terraform.tfvars.example terraform.tfvars
# แก้ไข terraform.tfvars ด้วย Azure subscription ID และ SSH key path ของคุณ

terraform init

terraform plan

terraform apply
```

Terraform จะแสดง public IP, คำสั่ง SSH และ URL แอปพลิเคชันหลังจากการ deploy สำเร็จ

## หลังการ Deploy

หลังจาก `terraform apply` เสร็จสมบูรณ์ ให้รอ 5--10 นาทีเพื่อให้ cloud-init ติดตั้ง Docker ดึง container images และกำหนดค่า nginx เสร็จสิ้น การดึง Docker image เป็นคอขวด — Juice Shop เพียงตัวเดียวมีขนาดประมาณ ~400 MiB

ตรวจสอบ health endpoint:

```bash
curl -s "http://$(terraform output -raw public_ip)/health" | jq .
```

ผลลัพธ์ที่คาดหวัง:

```json
{
  "status": "healthy",
  "component": "origin-server",
  "applications": ["juice-shop", "dvwa", "vampi", "httpbin", "whoami", "csd-demo", "dvga", "restaurant", "crapi"]
}
```

### การตั้งค่าฐานข้อมูล DVWA

DVWA ต้องมีการ initialize ฐานข้อมูลครั้งเดียว เปิด `http://<PUBLIC_IP>/dvwa/setup.php` ในเบราว์เซอร์แล้วคลิก **Create / Reset Database** ข้อมูลรับรองเริ่มต้นคือ `admin` / `password`

:::note
ฐานข้อมูล SQLite ของ VAmPI จะถูก initialize โดยอัตโนมัติผ่านสคริปต์ gunicorn entrypoint แต่ละ container จะรัน `db.create_all()` เมื่อเริ่มต้น ดังนั้นจึงไม่จำเป็นต้องดำเนินการด้วยตนเอง
:::

## การเชื่อมต่อกับคอมโพเนนต์ปลายทาง

หลังจากการ deploy คอมโพเนนต์อื่นๆ จะใช้ outputs ของ origin server เป็น inputs ของตน:

```bash
# CDN Simulator — ส่ง public IP ของ origin-server เป็น upstream
cd ../cdn-simulator/terraform
origin_ip=$(cd ../../origin-server/terraform && terraform output -raw public_ip)
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
origin_server   = "http://${origin_ip}"
origin_host     = "${origin_ip}:80"
EOF

# Traffic Generator — ส่ง F5 XC load balancer FQDN (ไม่ใช่ origin IP โดยตรง)
cd ../traffic-generator/terraform
cat > terraform.tfvars <<EOF
subscription_id = "your-subscription-id"
target_fqdn     = "your-xc-load-balancer.example.com"
EOF
```

| Output | คอมโพเนนต์ปลายทาง | ตัวแปร Input |
|---|---|---|
| `public_ip` | CDN Simulator | `origin_server` (พร้อมคำนำหน้า `http://`) |
| `public_ip` | CDN Simulator | `origin_host` (พร้อมคำต่อท้าย `:80` ไม่มี scheme) |
| `public_ip` | F5 XC Origin Pool | ที่อยู่ origin server |
